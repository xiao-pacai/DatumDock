# 安全策略

## 报告漏洞

请不要在公开 Issue 中披露可能导致数据泄露、任意文件删除、路径穿越、模型文件执行风险或项目备份篡改的问题。

在仓库公开前，请通过仓库所有者指定的私密联系方式报告，并提供：

- 受影响的版本或提交；
- 可复现步骤与最小示例；
- 影响范围；
- 可行的缓解建议（如有）。

维护者会确认收到报告、评估影响，并在修复完成后协调披露方式。

## 支持范围

项目当前处于预发布开发阶段。安全修复优先覆盖主分支和最近的可发布版本；不要将真实数据集、私有模型或凭证附在公开报告中。

## English Summary

Do not disclose vulnerabilities involving data exposure, arbitrary file deletion, path traversal, model-file execution, or backup tampering in public issues. Before the repository has a private security channel, contact the repository owner through their designated private method and include affected versions, reproducible steps, impact, and mitigations. Do not attach real datasets, private models, or credentials.
