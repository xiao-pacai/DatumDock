# DatumDock 现代视觉设计规范 v2（待实现）

> 状态：2026-07-19 已确认重新设计方向；本文件只定义目标视觉和实施门槛，当前不代表 GUI 已完成换肤。
>
> 优先级：本规范覆盖旧文档中以大面积暖灰、灰绿色、传统 Qt 工具栏和“莫兰迪即低对比”为主的视觉描述。业务交互仍以 `docs/UX.md` 为准。

## 1. 调整原因

现有设计方向虽然克制，但如果大量使用米白、灰绿、小字号、细边框和传统桌面菜单，容易形成“旧式管理软件”观感。DatumDock 的新目标不是简单提高颜色饱和度，而是同时更新布局节奏、组件形态、图标、层级、反馈和新手引导。

目标关键词调整为：

- 现代、轻快、清晰、友好。
- 像成熟开源工具，但不冰冷、不粗糙。
- 首页容易接近，标注工作台高效专业。
- 有鲜明品牌色，但画布和数据始终是主角。

## 2. 参考来源与边界

- [Scratch 编辑器导览](https://scratch.mit.edu/help/studio/tips/ui/tour-intro/)用于参考清晰分区、友好引导、大点击目标、圆润控件和图标化表达。
- [Scratch Foundation 开源 GUI](https://github.com/scratchfoundation/scratch-gui)用于理解其编辑器是完整组件系统，而不是一组孤立配色。
- [X-AnyLabeling 用户手册](https://github.com/CVHub520/X-AnyLabeling/blob/main/docs/en/user_guide.md)用于参考画布优先、工具栏、右侧标签/文件面板、快捷键和高频标注反馈。

参考不等于复制：

- 不使用 Scratch 猫、积木造型、其品牌蓝/橙精确色值、字体、插图或源代码。
- 不复制 X-AnyLabeling 的图标、品牌、界面截图、控件代码或像素级布局。
- DatumDock 保留自己的 `DD + DatumDock` 品牌、浅橙/浅蓝双 D、数据集主页和数据治理心智。

## 3. 总体策略：友好外壳 + 专业工作台

DatumDock 使用同一套设计令牌，但允许主页和标注页有不同密度：

| 场景 | 视觉重点 | 密度 | 参考倾向 |
| --- | --- | --- | --- |
| 数据集主页 | 欢迎、数据集卡片、快速开始、学习中心 | 宽松 | 更接近 Scratch 的亲和与清晰引导 |
| 标注工作台 | 画布、矩形框、标签、图片状态、快捷操作 | 紧凑 | 更接近 X-AnyLabeling 的直接和高效 |
| 设置/标签/模型管理 | 表格、表单、筛选和说明 | 中等 | 现代生产力工具，清晰优先 |
| 对话框/向导 | 单一决策、风险范围、进度 | 中等 | 低干扰、强层级、明确确认 |

主页可以活泼，但标注页不得变成儿童软件。标注页可以紧凑，但不得退回默认 Qt 灰色控件和密集旧菜单。

## 4. 颜色系统 v2

旧的暖灰/灰绿方案不再作为主视觉。v2 改为冷白与浅蓝灰底色、清晰白色表面、适度鲜明的品牌蓝，以及 Logo 的浅橙/浅蓝辅助色。

以下为首轮实现建议值，最终仍需通过实际 GUI 截图和对比度测试微调：

| Token | 建议值 | 用途 |

| `appBackground` | `#F4F7FC` | 主页和管理页面冷调背景，避免发黄。 |
| `surface` | `#FFFFFF` | 卡片、面板、表格和弹窗主表面。 |
| `surfaceSubtle` | `#EDF2F8` | 次级区域、筛选条、空状态底板。 |
| `surfaceHover` | `#E8F0FF` | 可点击行和卡片悬停。 |

### 颜色使用规则

- 品牌蓝只用于可交互重点，不铺满所有面板；首页允许较大蓝色区域，标注页只保留顶部条、激活工具和焦点。
- 浅橙与浅蓝主要服务 Logo、引导插图和少量提示，不能同时在每个按钮上竞争注意力。
- 标签颜色继续由 `LabelColorService` 独立管理，不使用品牌色判断类别，也不使用标签色表达系统状态。
- 待审核、已完成、有问题和异常同时使用文字与图标；状态色只做辅助。
- 普通正文、禁用文案和边框必须保持足够对比度，不能为了“柔和”而变成浅灰看不清。

## 5. 字体与信息层级

- Windows 首选 `Segoe UI Variable`，中文回退 `Microsoft YaHei UI`，再回退系统无衬线字体；不打包来源不明字体。
- 普通正文以 14px 为基准；紧凑表格可使用 13px，但关键操作、状态和标签选择器不得小于 13px。
- 首页大标题建议 28–32px，页面标题 20–24px，分区标题 16–18px，辅助说明 12–13px。
- 使用字重、字号和间距建立层级，不依赖全大写、过多彩色文字或细灰分隔线。
- 中英文切换后允许控件宽度自适应；不能以裁切中文、缩小字体或省略关键状态换取布局整齐。

## 6. 间距、圆角与表面

- 使用 4px 基础网格，常用间距为 8、12、16、24、32px。
- 首页外边距 24–32px、卡片间距 16–20px；标注工作台工具间距 6–8px、面板内边距 10–12px。
- 首页和教程卡片圆角 14–16px；按钮、输入框和管理面板圆角 8–10px；标注工具按钮圆角 8px。
- 主按钮高度 38–42px，普通按钮 34–38px，左侧工具按钮建议 40–44px 方形点击区。
- 使用 1px 清晰边框和非常轻的投影区分表面。禁止厚重阴影、玻璃拟态、浮雕、凹陷边框和大面积渐变。
- 面板通过背景层级和留白分区，不使用大量带标题的旧式 `QGroupBox` 边框包围所有内容。

## 7. 首页视觉

- 顶部使用完整 DatumDock 字标、清晰页面标题、搜索、设置和帮助；可以使用一条克制的品牌蓝顶栏或浅蓝品牌区，但不照搬 Scratch 顶栏。
- “新建数据集”是主页唯一持续突出的主按钮；数据集卡片使用大缩略图、名称、统计和进度，不堆放一排小图标按钮。
- 快速开始使用横向步骤或轻量进度卡，颜色更活泼；学习中心使用插图化卡片，但插图采用 DatumDock 自有抽象数据/标注主题，不使用卡通动物或 Scratch 角色。
- 数据集卡片悬停时轻微抬升、边框变蓝并显示主要操作；删除、归档等低频操作放入更多菜单。
- 空主页要像产品欢迎页，不像文件选择器：使用 Logo、简短说明、新建按钮和教程入口，不出现大面积路径输入框。

## 8. 标注工作台视觉

- 保留已确认的四区布局：顶部主操作栏、左侧标注工具、中央画布、右侧标注/图片面板。
- 顶部条使用扁平现代结构，当前数据集以明显的圆角下拉控件显示；导入和导出有图标与文字，低频操作收进更多菜单。
- 左侧工具栏使用统一圆角图标按钮：未选中透明、悬停浅蓝、激活品牌蓝/浅蓝背景。禁止默认 Qt 工具栏的灰色凹凸边框和密集竖分隔线。
- 中央画布使用稳定深色背景，图片周围只保留必要留白；选中框控制点、十字线和标签色清晰，不使用装饰性阴影。
- 右侧两个面板使用白色或浅灰表面、清晰标题行和轻分隔。当前标注与图片列表保持高信息密度，但行高不低于 34px。
- 图片状态以紧凑圆角徽标显示；当前图片使用蓝色选中边框和浅蓝底，不用黄色/绿色状态色代替选中态。
- 标注工作台不使用大幅插图、渐变背景或跳动动画；所有视觉资源都为速度和辨识服务。

## 9. 组件语言

### 按钮

- 主按钮：品牌蓝实心、白字、清晰图标，可用于“新建数据集”“开始导出”等单一页面主操作。
- 次按钮：白色或浅表面、边框、主文字；悬停进入浅蓝背景。
- 幽灵按钮：用于工具栏和卡片低频动作，默认无边框。
- 危险按钮：只在确认区域使用红色，不把红色垃圾桶长期放在主操作旁。

### 输入、筛选与下拉

- 高度统一，圆角 8–10px，使用可见边框和品牌色焦点环。
- 搜索框使用搜索图标、清除按钮和可读占位文案；筛选使用可移除的圆角 chip，不堆叠多行传统复选框。
- 下拉菜单项目具有图标、主文字、可选说明和快捷键列；当前项使用勾选与浅蓝背景。

### 表格与列表

- 表头使用浅灰蓝背景和中等字重，不用深灰渐变表头。
- 行悬停、选中、键盘焦点和状态分别表达；斑马纹仅在确实提升长表可读性时使用。
- 空表格用简短说明和一个下一步操作，不显示一块旧式空白网格。

### 对话框

- 标题明确说出任务，正文先给结论再列影响；危险确认列出数量和不可恢复范围。
- 按钮顺序保持 Windows 习惯，主按钮与取消按钮位置稳定；不使用“是/否”替代具体动作名称。
- 导入、推理、重命名和导出进度对话框展示当前阶段、总进度、取消与错误详情。

## 10. 图标与插图

- 图标使用 DatumDock 自有的圆角线性风格，统一 2px 左右视觉笔画、圆角端点和 20/24px 网格。
- 激活工具可使用填充或双色变体；普通状态保持单色，避免每个图标都是不同鲜艳颜色。
- 图标必须在浅色表面和深色画布旁均可辨识，并提供 hover、active、disabled 与 high-DPI 资源。
- 首页插图使用图片、矩形框、标签、数据分组、模型等抽象主题；形状可以更友好，但不使用幼儿化吉祥物。
- 不直接使用 Scratch、X-AnyLabeling、系统截图或第三方商业图标包中的受限资源。

## 11. 动效与反馈

- 常规 hover/press/展开使用 120–180ms；页面切换可使用 180–220ms 淡入或位移，且尊重系统“减少动态效果”。
- 数据集卡片只做 1–2px 视觉抬升和边框变化，不使用大幅缩放、弹跳或持续发光。
- 工具切换、自动保存、后台任务和状态更新在原位置给出即时反馈，不频繁弹 toast。
- 成功提示短暂且克制；错误提示保留到用户处理，并提供可操作下一步。
- 教程插图或短动画不得自动播放声音，长动画提供暂停。

## 12. 现代感禁止项

以下模式视为视觉回归失败：

- 默认 Qt/Fusion 灰色外观、立体按钮、凹陷输入框和密集工具栏分隔线。
- 大面积米黄、灰绿、棕灰背景造成旧式办公软件观感。
- 以“文件 / 编辑 / 视图 / 工具 / 帮助”传统菜单行承载全部主要操作，用户必须层层寻找高频功能。
- 12px 以下常驻正文、过窄行高、按钮文字被裁切、图标锯齿或不同来源风格混杂。
- 每个区域都加边框、每个标题都放进 `QGroupBox`、每个按钮都同时显示高饱和颜色。
- 大面积渐变、玻璃拟态、厚阴影、霓虹科技风、过度圆润到像儿童玩具。
- 复制 Scratch 的品牌元素或让界面看起来像 X-AnyLabeling 的换色版本。

## 13. Qt 实施边界

- `ThemeService` 提供语义 token，QSS 只消费 token 派生值；业务控件禁止散落颜色、圆角和字号常量。
- 优先使用 Qt 样式、布局和自有 SVG 图标实现，不通过大图截图伪造控件。
- 保留 Windows 原生窗口边框与最小化/最大化/关闭行为；品牌顶栏放在应用内容区，除非后续单独验证无边框窗口的拖动、缩放、DPI 和辅助功能。
- 为按钮、输入框、菜单、表格、列表、卡片、徽标、工具栏、对话框和空状态建立可复用组件层，不在每个页面重复 QSS。
- 所有组件验证 100%、125%、150%、200% DPI；SVG 图标按设备像素比渲染，文本不使用固定像素高度裁切。
- 主页和标注工作台建立固定截图测试场景，至少覆盖中文/英文、空状态/有数据、正常/hover/selected/disabled、100%/150% DPI。

## 14. 视觉验收闸门

- [ ] 首次打开不再呈现默认 Qt 灰色、米黄灰绿或传统 IDE/文件管理器观感。
- [ ] 主页在 5 秒内能看出“已有数据集”“新建数据集”“快速开始/教程”三类入口，且主次明确。
- [ ] 标注页在 5 秒内能看出当前数据集、导入/导出、矩形框、AI 标注、画布、当前标注和图片状态列表。
- [ ] 首页友好活泼，但标注工作台保持专业、紧凑且画布优先。
- [ ] 主按钮、次按钮、工具按钮、危险按钮和禁用状态一眼可区分。
- [ ] 组件在中文/英文及 100%–200% DPI 下无裁切、重叠和模糊图标。
- [ ] 键盘焦点、hover、选中、图片状态和错误不只依赖颜色。
- [ ] 首页、标注工作台、标签管理、模型管理、设置、导出向导和危险确认使用同一组件语言。
- [ ] 对照截图未出现本文件“现代感禁止项”中的任一回归。
- [ ] 所有参考均为设计原则借鉴，发布包不包含 Scratch 或 X-AnyLabeling 的受限品牌与图标资产。

## English Summary

DatumDock visual design v2 combines Scratch-inspired friendliness, clarity, rounded controls, onboarding, and approachable color with X-AnyLabeling-inspired canvas focus, compact tools, right-side information density, and direct annotation feedback. The home page is spacious and welcoming; the annotation workspace is tighter and professional. Cool light backgrounds, white surfaces, a clear DatumDock blue, the existing light-orange/light-blue DD identity, modern typography, consistent rounded icons, restrained motion, and a dark canvas replace the old warm-gray/gray-green Qt-like appearance. This is a documented target, not an implemented redesign, and no third-party brand assets or pixel-level layouts may be copied.
