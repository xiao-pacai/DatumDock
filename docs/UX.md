# DatumDock 交互与界面规范（MVP）

## 1. 主窗口布局

```text
┌─────────────────┬──────────────────────────────────┬─────────────────┐
│ 项目与数据集    │              图像画布             │ 标签与属性      │
│ - 工作区切换    │                                  │ - 标签搜索/列表 │
│ - 项目/数据集树 │    缩放、平移、矩形标注与选中态   │ - 标签说明      │
│ - 数据集池筛选  │                                  │ - 当前标注属性  │
│ - 搜索与样本    │                                  │                 │
├─────────────────┴──────────────────────────────────┴─────────────────┤
│ 状态栏：图片序号、尺寸、缩放比例、保存状态                         │
└───────────────────────────────────────────────────────────────────┘
```

## 2. 画布规则

- 鼠标左键拖拽：在“创建框”状态新建矩形；在选中框内拖拽则移动该框。
- 框被选中时显示八个缩放控制点。
- 鼠标滚轮：以鼠标位置为中心缩放。
- 按住空格拖拽：平移画布。
- 框颜色取自类别颜色，选中状态应有明显高亮。

## 3. 多项目与数据集池体验

- 左栏顶部显示当前工作区，并提供最近项目和“打开工作区”入口。
- 项目/数据集树支持展开、折叠和快速切换；当前项目与数据集应始终清晰可见。
- 同时打开多个项目时，允许以标签页或最近列表快速切换；切换不应重新导入或扫描无关数据集。
- 左栏显示当前数据集的全部样本，并支持按来源、标注状态、标签和文件名筛选。
- 样本列表支持至少一万张图片，采用虚拟列表与延迟缩略图：首屏优先显示、滚动时按需加载，并在后台显示缩略图生成进度，不因全量加载卡住界面。
- 每个样本显示缩略图、文件名、标注数量和状态：未标注、待人工复核、已复核、有问题、异常。
- 网格工具栏提供“显示标注预览”开关；开启时缩略图叠加矩形框、标签色和别名，关闭时只保留原图与状态标记。
- 顶部工具栏提供“新建数据集”“导入数据”“整理文件名”“自动标注”“导出数据集”“管理项目标签集”“管理项目模型”等主要入口；两个“管理”入口打开独立项目级页面，其他操作目标默认是当前数据集。
- “新建数据集”向导先选择“空数据集”或“从已有数据集创建”。模板模式显示将复用的标签集、命名规则和导入规范，并明确不会复制图片和标注。
- 数据集右键菜单提供“比较标签兼容性”“复制到数据集”“移动到数据集”“合并到此数据集”。不兼容目标显示差异入口而非执行按钮。
- 导入向导允许选择 JPG/JPEG、PNG、BMP、WebP、TIFF 等常见静态图片；MVP 统一转换为 PNG，并显示格式、数量、重复项、失败项和可取消的后台进度。
- 导出采用一次性分步向导：选择目标训练模型/格式 → 选择候选范围 → 设置比例与种子 → 预览统计 → 选择输出位置 → 执行并展示结果。
- 切换模型格式时，向导动态显示该格式所需参数、支持的标注类型和输出示例；不支持的样本或标注要在执行前明确提示。
- 导出前预览显示 train/val/test 的样本数、标签框数量和相似组约束造成的比例偏差；存在问题时明确提示，而不是自动忽略。
- 数据集工具栏提供“相似图片检查”入口，显示高置信度相似组和待确认候选；用户可确认相似组、移出误报图片或忽略候选，图片不会被自动删除。

## 4. 视觉风格：莫兰迪色系

- 视觉目标是“轻盈、安静、专业”：默认使用明亮小清新的浅色主题、充足留白和低饱和莫兰迪色，而非高饱和科技风、厚重阴影或大面积深色界面。画布专注于图片和标注，管理界面专注于信息层级。
- 参考 X-AnyLabeling 的高效三栏标注工作流、直接反馈和紧凑但不拥挤的信息密度；不得复制其品牌、图标、源代码或让用户误以为这是 X-AnyLabeling。

### 品牌使用

- 应用主窗口左上角固定设置品牌区：常规侧栏显示完整 `DD + DatumDock` 字标，作为工作区、项目与数据集导航的起点；点击它返回当前工作区概览，不应触发数据修改。
- 当用户将侧栏折叠到窄宽度时，只显示浅橙色与浅蓝色的 `DD` 标记；鼠标悬停显示 `DatumDock` tooltip，展开侧栏后恢复完整字标。不得将完整字标挤压到难以辨认的尺寸。
- 欢迎/空工作区页、项目创建向导首屏与“关于 DatumDock”页面使用完整字标；普通内容页不重复放置大 Logo，避免挤占数据管理与标注空间。
- 所有应用内展示统一使用 `assets/brand/datumdock-wordmark-v3.png`；在 GUI 资源打包阶段将它纳入 Qt 资源。后续的小尺寸窗口图标、托盘图标和安装包图标从同一 `DD` 标记派生，不以文字截图替代。
- 品牌字标仅用于产品身份，不可作为危险操作、状态颜色、标签类别颜色或可点击的主要操作按钮；始终提供文字标题和无障碍名称，不让 Logo 成为唯一的导航线索。

### 设计令牌

| 用途 | Token | MVP 建议值 | 使用规则 |
| --- | --- | --- | --- |
| 应用背景 | `appBackground` | `#F7F6F2` | 窗口边缘与大面积留白，避免纯白刺眼。 |
| 基础表面 | `surface` | `#FFFFFF` | 卡片、表格、对话框和输入区。 |
| 弱表面 | `surfaceSubtle` | `#EEF1EE` | 侧栏分组、悬停和次级区域。 |
| 面板表面 | `panel` | `#DFE7E4` | 左右栏标题、工具栏和信息分组。 |
| 主强调 | `accent` | `#78978C` | 主按钮、焦点、选中边线和进度；只用于关键操作。 |
| 次强调 | `accentSoft` | `#DDE9E1` | 选中项填充、成功背景和弱提示。 |
| 次要点缀 | `roseSoft` | `#D8B5AE` | 次级提醒或图表点缀，不承担错误语义。 |
| 主文字 | `textPrimary` | `#35403C` | 正文与标题，禁止使用生硬纯黑。 |
| 次文字 | `textSecondary` | `#68736E` | 辅助说明、统计和禁用前说明。 |
| 错误 | `danger` | `#B36F68` | 永久删除、错误和阻断提示；必须同时配图标和文字。 |

- 标签颜色是项目数据的一部分，独立于上述应用主题 token；不能为了统一主题而降低标签之间的可区分度，也不能将标签颜色错误地用作“保存成功/删除失败”等系统语义。
- 选中项采用“浅色填充 + 2px 主强调边线/焦点环”；悬停只改变弱表面或边线，不做跳动、闪烁和明显位移。
- 界面仅使用轻微 1px 分隔线与小幅阴影区分层级。卡片圆角建议 10px，按钮与输入框圆角建议 8px；禁止玻璃拟态、强渐变和悬浮夸张阴影。

### 版式、文字与反馈

- 采用 8px 间距基准：控件内边距 8–12px、相邻表单字段 12–16px、独立信息分组 24px。不要把每个区域都塞满控件；低频设置放入折叠区或二级页面。
- 默认正文 14px、辅助文字 12px、页面标题 20–24px、分组标题 16px；表格行高和工具栏高度应保证鼠标操作舒适，并随系统缩放比例正确放大。
- 缩略图卡片保持统一比例、圆角裁切和清晰状态角标；只有鼠标悬停或键盘焦点时显示低频操作，避免网格长期堆满按钮。
- 图片画布使用中性暖灰底，不干扰原图颜色；未选中框使用标签色轮廓和低透明填充，选中框额外显示更粗边线、控制点和标签浮层。画布的拖拽、缩放、框选反馈必须即时，不能等待动画结束。
- 主操作在每个页面最多保留一个明显的实心强调按钮；危险操作永远不与普通操作同等突出，必须经过确认对话框。空状态应有简洁图标、下一步说明和一个主入口。
- 键盘焦点、加载、成功、错误、禁用、未保存和复核状态都必须有文字或图标补充。悬停帮助使用圆形问号图标，不用大段常驻说明破坏页面呼吸感。
- 动画仅用于界面状态过渡，时长建议 120–180ms，并尊重系统“减少动态效果”偏好；不得延迟绘制、缩放、拖拽、切图或快捷键反馈。

### 图标资产

- DatumDock 的产品图标、导航图标、工具栏图标、空状态插图和安装包图标均作为自有项目资产生成并保存在 `assets/icons/`；其中应用图标从品牌 `DD` 标记派生。不得直接复制 X-AnyLabeling 或其他产品的品牌标识、图标文件或视觉资产。
- 图标统一采用圆角、简洁线性、低饱和单色/双色风格，默认 20px 或 24px 视觉网格；需要强调时使用主题语义色，不使用与项目标签混淆的颜色。
- 每个图标必须有清晰文字 tooltip、可访问名称和禁用/悬停/焦点状态。仅图标按钮在常用场景下仍须能通过 tooltip 和快捷键说明被理解。
- 首批至少覆盖：应用图标、工作区/项目/数据集、导入、导出、标签、模型、自动标注、筛选、搜索、设置、帮助、保存、撤销/重做、删除、回收站、上一张/下一张、缩放、适配画布和状态提示。
- 用户后续指定某个图标需要修改时，以对应源资产为唯一修改目标；更新后需生成目标尺寸并在亮色主题、悬停、禁用和高 DPI 下进行视觉回归。

## 5. 应用语言与设置

- 主菜单或应用工具栏提供“设置”入口；设置页包含“界面语言”下拉框：`简体中文` 与 `English`，默认选中简体中文。
- 选择语言后立即刷新当前可见的主窗口、菜单、工具栏、状态栏、标签/模型管理页面和对话框；无需重启应用。
- 设置页提供当前语言效果预览或说明文字，帮助用户确认切换结果。
- 语言切换不得改变标签名称、中文别名、描述、数据集名、图片文件名或模型配置；这些内容始终按项目原值显示。
- 标签选择器的“中文别名 / 英文训练名 / 双语”显示方式属于标注显示偏好，与系统界面语言独立设置。
- 设置页“数据管理”分区提供“回收站少量样本阈值”和“默认导出比例”。每项右侧使用圆形问号图标，悬停显示该设置的影响、建议值和示例。

### 快捷键设置

- 设置页的“快捷键”分区按“项目与文件、浏览、画布、标注编辑、标签、自动标注、导出”分组显示可配置操作。
- 每行显示操作名称、当前绑定、默认绑定和说明；可用搜索框按名称或当前按键过滤。
- 点击绑定字段后进入录入状态，用户直接按下组合键；界面显示标准化后的组合，例如 `Ctrl+Shift+S`。
- 发现冲突时在该行显示冲突操作，并提供“取消”“替换原绑定”“选择其他按键”；系统保留组合显示为不可用。
- 底部提供“恢复当前项默认值”“恢复分组默认值”和“恢复全部默认值”。所有成功修改立即在已打开界面生效。

## 6. 快捷键

| 快捷键 | 行为 |
| --- | --- |
| `Ctrl+O` | 导入图片或文件夹到数据集池。 |
| `Ctrl+S` | 立即重试保存当前图片标注（通常编辑后已自动保存）。 |
| `Ctrl+E` | 打开数据集导出向导。 |
| `A` / `D` | 上一张 / 下一张图片。 |
| `Delete` | 删除当前选中标注。 |
| `Ctrl+Shift+Delete` | 删除当前图片及其全部受管关联信息。 |
| `Esc` | 取消正在创建的标注或取消选择。 |
| `Ctrl+Z` / `Ctrl+Y` | 撤销 / 重做（可在 MVP 后半段实现）。 |

## 7. 标签集与标签选择器

- 标签集由项目级“管理项目标签集”入口打开的独立页面维护；标注侧栏只负责选择和查看标签。
- 标签管理页面上方是标签总表，默认列为：颜色、英文训练名、中文别名、描述、同义词、类别 ID、状态、使用次数和操作；支持搜索、排序、筛选、添加、编辑、归档，并在每行提供“查看图片”操作。
- 每个活动标签以独立颜色色块显示。新建时自动分配未使用的高辨识度颜色；编辑颜色时使用调色板和颜色选择器，并显示与已有标签的冲突提示。
- 标签颜色一致用于画布框线和半透明填充、标签选择器色点、标签检查集合的标记和自动标注建议框。选中、错误和待确认状态仍使用额外的边线、图标或纹理，避免只依赖颜色。
- 选择表格行后，下方或右侧详情区展示完整描述、同义词、创建信息、使用范围和修改入口；表格应支持直接编辑安全字段。
- 标签选择器顶部始终有搜索框；支持用中文别名、英文训练名、同义词或描述关键词定位标签。
- 默认条目格式为“`中文别名` · `english_name`”，下一行显示简短描述；不应只显示英文名称。
- 选中条目后，右侧说明区显示完整中文描述、英文训练名、类别 ID 和同义词，帮助用户在相似物品间做判断。
- 当标签数超过常用屏幕可见范围时，提供“最近使用”“常用置顶”“全部标签”分组与键盘上下选择；所有标签仍能通过搜索立即到达。
- 若标签被归档，已有标注仍能正确显示；新建标注默认不再提供该标签。
- 用户编辑英文训练名或类别 ID 时，弹出迁移确认对话框：显示旧值、新值、受影响数据集、样本数和标注数；确认前不写入任何数据。
- 迁移进行中显示总进度、当前文件和失败信息；完成后刷新标签表、画布、筛选结果和当前标注文案。
- 点击“查看图片”进入标签检查视图，默认汇总当前项目所有数据集中含该标签的图片；顶部可切换“全部数据集 / 指定数据集”。
- 标签检查视图以网格或列表展示缩略图、文件名、所属数据集、目标标签框数量和标注状态；点击图片会打开其原始标注页并高亮目标标签框。
- 检查视图是筛选结果而非新文件夹或新数据集，关闭后不会创建副本；样本被删除或标签变更时列表实时刷新。

## 8. 文件命名与样本删除

- “整理文件名”打开当前数据集的命名规则对话框。用户可设置前缀、起始编号、补零位数、命名模板和是否保留原扩展名，并选择“全部样本”或“当前筛选结果”。
- 输入规则时实时显示旧文件名 → 新文件名预览，标出重名、非法字符、编号空缺和无法写入的文件；存在错误时“执行重命名”不可用。
- 确认重命名后显示进度；完成后文件列表、画布标题和统计数据立即使用新名称。用户无需手工处理同名 LabelMe JSON 或 JSON 中的图片路径。
- 标注页面的“删除当前图片”使用显著危险色按钮，并与删除标注框的 `Delete` 键区分。确认对话框必须列出图片名、标注框数和将被一起删除的关联项。
- 删除对话框根据设置中的少量样本阈值提供“移入软件回收站”和“永久删除”两种明确选项，并显示样本数量和受影响关联项；批量删除时允许直接选择永久删除。
- 删除成功后自动跳转到下一个可用样本；当前图片、配套 JSON、索引项、缩略图、缓存和待确认自动标注一并移入回收站或永久消失，标签集和其他图片不受影响。

## 9. 模型库与自动标注

- “管理项目模型”打开独立模型管理页面。页面上方表格展示：模型名称、格式、文件大小、任务类型、输入尺寸、类别数、状态、导入时间、标签映射状态和操作。
- 用户可点击“导入模型”选择 ONNX、PT 等文件。导入进度包含复制、解析、校验和配置确认四个阶段；解析结果以可读方式展示，而不是静默完成。
- 自动探测成功时，配置页展示模型的输入尺寸、预处理、输出解码、类别表、置信度阈值、NMS 阈值及项目标签映射；用户可确认或修改需要配置的字段。
- 不支持或探测不完整的模型显示明确原因、缺失字段和可采取的下一步；禁止提供看似可用但实际未知的“自动运行”按钮。
- 更新模型时，用户在同一模型条目中导入替换文件；新文件验证成功前，旧模型保持可用。删除模型前显示受影响自动标注来源提示，确认后只移除该项目的模型文件和配置。
- 在数据集页面选择“自动标注”后，用户选择已验证模型、处理范围（当前图片、全部图片、全部未标注图片）和阈值。预测框直接保存为带模型来源、置信度和图片级“待人工复核”状态的项目标注；用户可在画布上调整漏标、误标、标签和框位置，然后将整张图片标记为已复核或有问题。完成问题处理后标为已复核会清除“有问题”状态，不能自动覆盖人工标注。
- 推理前优先使用可用 GPU；首次只能使用 CPU 时弹出说明，提供“继续使用 CPU”和“查看 GPU 配置方法”入口。
- 双击矩形框直接打开标签切换控件，其他画布编辑手势以 X-AnyLabeling 的常见操作习惯为参考。

## 10. 项目备份与数据集间转移

- 文件菜单提供“导出项目备份”和“导入项目备份”。备份向导明确说明模型二进制不会包含在项目包内，需要单独传递；模型配置会随项目保存。
- 导入备份时先显示格式和完整性校验进度；校验通过后才显示新项目名称和保存位置，失败时显示原因并不创建项目。
- 数据集操作菜单提供“复制到数据集”和“移动到数据集”。目标列表仅显示标签集完全一致的数据集；不一致数据集以禁用状态并显示原因。

## 11. 标签集与数据集合并

- 标签管理页面的“比较标签集”对话框使用双栏/表格展示：训练映射一致、显示信息不同、仅源存在、仅目标存在和阻断冲突。
- “合并标签集”按钮仅在没有阻断冲突时可用。用户选择目标标签集为主版本后，预览将补充的标签和将保留的显示信息。
- 数据集合并向导显示源/目标名称、样本数、标签训练签名、完全重复图片数和“复制/移动”选择；发现标签不兼容时禁止继续。
- 合并进度页显示已处理、跳过重复、失败和待处理数量；结束后刷新目标数据集网格、状态统计和标签检查结果。

## 12. 自动保存与待处理状态

- 标注创建、移动、缩放、删除、标签修改、撤销和重做后立即触发原子自动保存；状态栏显示“正在保存”与“已保存”。
- 自动保存失败时，当前文档进入待处理状态；切换图片或关闭窗口时，提供“重试保存 / 放弃内存修改 / 取消”三个选择。
- `Ctrl+S` 始终可用于立即重试保存；成功后状态栏显示“已保存”，失败信息必须保留到用户处理。

## 13. 无内容状态

- 未打开项目时，显示“创建项目或打开已有项目以开始管理数据集”。
- 项目为空时，画布显示“导入图片或文件夹以建立数据集池”。
- 文件夹不含支持图片时，说明没有找到图片并列出支持的格式。

## 14. X-AnyLabeling 导入与交换

- 文件菜单提供“导入 X-AnyLabeling 标注目录”和“导出为 X-AnyLabeling 目录”。两项均显示格式说明链接/问号帮助：交换目录使用图片、同名 LabelMe JSON 与 `labels.txt`，不等同于 YOLO 训练集导出。
- 导入向导按“发现图片 → 配对 JSON → 可编辑矩形 → 已保留但不可编辑内容 → 错误/缺失文件”展示统计与可展开的文件级报告。用户必须能在导入前看出哪些 shape 不能在当前版本编辑。
- 打开含已保留 shape 的图片时，右侧属性面板显示兼容性提示和 shape 数量；已保留内容在画布中以低干扰样式显示（若能安全渲染），不能被误认为可编辑矩形。保存前不需要用户额外选择“保留”，这是默认且不可关闭的安全行为。
- 导出向导说明它会使用项目标签的英文训练名，而中文别名仅用于 DatumDock 界面；完成页显示导出路径、图片/JSON 配对数量、保留 shape 数量及失败项，并提供“在资源管理器中打开”。

## English Summary

The main GUI contains workspace/project/dataset navigation, a dataset-pool sidebar, an image annotation canvas, and a label/property panel. Its default bright, fresh Morandi design system uses centralized semantic tokens, generous 8px-based spacing, restrained corners/shadows, and direct canvas feedback; it takes workflow inspiration from X-AnyLabeling without copying its brand. The full DatumDock wordmark is fixed at the top of the normal sidebar, while its `DD` monogram is used in the collapsed sidebar and later small application-icon contexts; welcome and About pages use the full wordmark. Every active label has a unique color shown consistently in the table, picker, canvas, inspection collection, and auto-annotation suggestions; states also use non-color cues. It can import X-AnyLabeling directories and export directly reopenable image-plus-LabelMe-JSON directories, preserving unsupported shapes as read-only compatibility payloads. Application settings offer immediate Simplified Chinese/English UI switching and a searchable shortcut editor with conflict detection.
