# DatumDock 交互与界面规范（MVP）

## 0. 最新入口规范（步骤二入口已实现）

用户入口以 [内部数据集主页与存档式管理方案](DATASET_LIBRARY.md) 为准：DatumDock 每次启动先显示存档式数据集主页，用户不选择工作区、项目根目录或外部保存位置。步骤四已在真实图片池上接入数据集标签、矩形编辑、立即自动保存、图片级复核和标签检查；模型与导出交互仍按本文规范等待后续接入。

### 1.1 顶部标题与主操作栏

- 顶部区域参考 X-AnyLabeling 的高频操作组织方式，但使用 DatumDock 自有品牌、图标和现代视觉 v2，不复制其品牌资产。
- 左侧先显示独立返回主页按钮，再显示不可点击的 DatumDock 静态品牌；中部突出当前数据集名称。点击数据集名称或下拉箭头，打开支持搜索的快速切换菜单，列出最近数据集并提供“查看全部数据集”。
- 快速切换数据集不得弹出外部目录选择器。切换前等待当前图片自动保存；保存失败时阻止切换并提供重试、放弃内存修改或取消。
- 常驻高频操作至少包含“导入图片”和“导出”。“导入图片”把外部图片复制到当前受管池；“导出”打开菜单或向导，区分训练数据集导出、X-AnyLabeling 交换目录和数据集备份，避免把不同含义都写成模糊的“导出图片”。
- 标签管理、模型管理、设置和低频数据治理操作可放在标题栏右侧或“更多”菜单。标题栏不得堆满文字按钮；窗口较窄时，低频操作自动收进菜单，高频操作仍可到达。
- 顶部操作必须使用集中式 `action_id` 和用户可配置快捷键；仅图标按钮必须提供中英文 tooltip 和无障碍名称。

### 1.2 左侧标注工具栏

- 左侧使用窄竖向工具栏，主要放“对画布做什么”的工具，而不是图片列表或数据集树。
- MVP 常驻工具至少包含：选择/编辑、创建矩形框、平移、适配窗口、撤销和重做。放大、缩小和缩放比例必须有明确入口；待本轮交互整改实施后，普通滚轮不再负责缩放，而用于竖直浏览图片。
- “AI 标注”作为带下拉菜单的明显操作，提供“标注当前图片”“标注全部图片”“标注全部未标注图片”；执行前选择模型、阈值和运行后端，批量任务显示进度与取消入口。
- 当前激活工具必须有图标、文字 tooltip、选中背景和键盘焦点，不只依靠颜色。危险操作如删除图片不得与普通选择工具并排造成误触。
- 工具栏可折叠为更窄模式，但矩形框与 AI 标注在常用屏幕尺寸下不得被隐藏到难以发现的多层菜单中。

### 1.3 中央画布

- 中央画布获得窗口最大可用空间；面板拖动或窗口缩放时，优先保证当前图片和标注框可操作。
- 画布显示当前图片、矩形框、选中控制点、标签颜色和必要的自动标注置信度；列表选择和画布选择保持双向同步。
- 图片切换后保持用户选择的缩放策略：适配窗口模式自动适配，手动缩放模式按产品设置决定是否保留比例；不得因为右侧列表刷新让画布明显跳动。

### 1.4 右侧标注与图片面板

- 右侧面板上下分区，上方显示“当前图片的标注”，下方显示“图片列表”；两区之间使用可拖动分隔条，并允许分别折叠。
- 当前图片标注列表的每一行至少显示标签颜色、中文别名、英文训练名和选中状态；点击列表项选中并定位画布框，点击画布框也同步选中列表项。
- 当前图片标注列表提供搜索、切换标签和删除所选框的入口。单击标注项会选中对应画布框；列表或画布具有操作焦点时，按 `Delete` 均删除这一条矩形标注。双击标注项或画布框可快速切换标签；确认改派严格只产生一个历史节点、文档版本和保存请求。双击过程中建立的移动快照必须清理，弹窗关闭后的配对释放不得触发虚假移动保存。
- 右侧列表中的 `Delete` 只删除当前图片里选中的标注实例，不删除数据集级标签定义，也不影响其他图片中的同类标注。输入框、搜索框或标签编辑表单具有文本焦点时，`Delete` 只执行正常文本删除，不得误删标注。
- 从右侧列表删除标注与从画布删除使用同一命令：只产生一个撤销历史节点和一次自动保存，列表、画布、框数和图片状态同步刷新；`Ctrl+Z` 可以恢复。对待复核图片执行删除属于一次有效人工编辑，保存成功后自动变为已完成；删除最后一个框时以“已完成 + 零框”表达人工确认的无目标图片。
- 图片区支持列表/网格切换、文件名搜索、状态筛选和标签筛选。列表行至少显示缩略图、文件名、标注框数量，以及固定在行末的图片级状态徽标。
- 状态徽标始终使用“文字 + 图标/形状”，颜色只做辅助；即使文件名很长，状态也固定在最右侧并保持可读。当前图片使用独立选中边框，不用状态颜色代替选中态。
- 图片列表采用 SQLite 分页、虚拟化和延迟缩略图，在一万张图片时不得一次创建全部行或加载全部缩略图。

### 1.5 图片级状态文案

右侧列表中的复核状态以整张图片为单位，不属于某一个矩形或某一种标签。下一轮整改后只保留两个用户可见复核状态：

| 用户可见状态 | 内部含义 | 进入条件 |
| --- | --- | --- |
| 待复核 | 模型自动标注已经写入，但尚未由人工完整检查。 | 任意模型结果改变该图片的标注后进入。 |
| 已完成 | 人工创建或有效编辑的标注，或人工检查模型结果并明确确认没有问题。 | 纯人工画框默认进入；待复核图片发生一次有效人工编辑，或通过确认操作进入。 |

- 新导入但尚无任何标注的图片不显示第三种复核状态；“无标注”作为可筛选的数据条件存在，不属于状态徽标。
- 缺图、损坏 JSON、未知标签和恢复失败继续显示独立的健康/诊断提示，不属于复核状态，也不增加第三种状态。
- 纯人工创建第一个矩形后，整张图片默认标为“已完成”。人工继续编辑这类图片时保持“已完成”。
- 模型自动标注或重新运行模型并改变标注后，整张图片必须标为“待复核”，即使此前已经完成。
- 待复核图片发生一次有效人工标注编辑后，在该编辑成功保存时自动变成“已完成”。有效编辑包括创建、移动、缩放、删除、换标签，以及实际改变标注文档的撤销/重做。
- 选择框、打开标签小窗但取消、缩放或平移视图、搜索、筛选和切换面板不改变标注文档，因此不触发自动完成。
- 如果用户检查模型标注后无需修改，可通过“确认已完成”按钮或可配置快捷键完成复核。
- “已完成（无目标）”不再作为单独状态。零矩形图片经人工明确确认后同样显示“已完成”，是否为负样本由零框事实表达。
- “有问题”“未复核”“待审核”不再作为用户可选复核状态；原来的有问题图片在迁移时安全归入“待复核”。
- 状态变更立即写入数据集索引，并同步更新右侧列表、主页复核进度、筛选统计和导出候选预览。

> 状态：本节双状态规范已由 schema v3 和正式工作台实现；步骤四早期五状态截图仅作历史证据。

## 2. 画布规则

- `R`：进入一次性矩形创建模式。成功创建一个矩形后自动返回选择/编辑模式，不保持连续画框状态。
- 进入矩形创建模式时，默认标签取当前数据集在本次应用会话中最近一次成功使用的活动标签。用户从 `test1` 切换并成功使用 `test2` 后，下一框默认选中 `test2`。
- 一次性矩形正式支持两种等价输入：按下左键拖拽并释放完成；或第一次单击确定一个角点、移动鼠标预览矩形、第二次单击确定对角点并完成。
- “单击两次”是两个位置上的两次独立单击，不是双击事件。空白图片区域在矩形创建模式中使用两次单击画框；双击已经存在的矩形仍打开“更改标签”小窗，两者不得互相误触发。
- 第一次单击后显示锚点、动态矩形轮廓、宽高和跟随指针的辅助线；第二个角点可位于第一个角点的任意方向，保存前统一规范化为左上/右下像素坐标。
- 两次单击形成零面积矩形时不得创建、保存或改变复核状态，并保留第一个锚点供用户重新指定有效对角点；按 `Esc` 可随时清除锚点并返回选择/编辑模式。
- `Esc`：取消尚未完成的矩形并返回选择/编辑模式；不得产生历史节点、自动保存或空标注。
- 在选择/编辑模式中，鼠标左键点击框用于选中，在框内拖拽用于移动该框。
- 框被选中时显示八个缩放控制点。
- 本节所说的“指针样式”专指 Windows/Qt 显示的实际系统鼠标指针图标，不改变矩形标注本身的边框、填充、标签文字或控制点样式。
- 选择/编辑模式下，指针命中矩形框内部的可移动区域时显示四向移动指针；按下拖动期间保持该样式，释放或取消后按新的悬停位置重新判断。
- 指针命中左/右控制点时显示水平缩放指针，命中上/下控制点时显示垂直缩放指针；左上/右下使用同一对角缩放指针，右上/左下使用另一对角缩放指针。指针方向必须与屏幕上的实际拖动方向一致。
- 鼠标指针反馈的输入优先级为：中键平移的闭合手形、矩形创建指针、缩放控制点、框内四向移动、普通箭头。按键释放、`Esc`、切图、工具切换、标注删除或失去画布焦点后不得残留旧指针。
- 只读兼容 shape、损坏/恢复锁定文档或其他不可编辑对象不显示移动/缩放指针；指针图标变化只是视觉提示，不创建历史、不触发保存或复核状态变化。
- 图片后方的中央底板使用冷调浅灰蓝，不再采用大面积深色；图片边缘使用克制的 1px 边线或等价层级，在白色图片上仍能看清真实边界，但不使用厚阴影。
- 矩形创建模式下，鼠标可在中央底板按下、释放或完成两次单击。落在底板上的点实时吸附到最近的图片边或角，动态矩形贴着图片边缘显示；指针回到图片内后继续使用实际图片坐标。
- 若拖拽从图片内越过边缘，超出的横纵坐标分别钳制到图片范围；若从底板开始再拖入图片，起点固定在最近边缘。两个点位于图片相对两侧时可形成贯穿到两侧边缘的矩形。
- 底板吸附只在矩形工具且图片可编辑、活动标签可用时生效。选择模式点击底板可清除当前选择但不创建框，中键拖拽仍优先平移，点击中央画布之外的任何 UI 区域都不能触发吸附。
- 吸附预览应提供边缘高亮或吸附点等轻量反馈。A0.5 十字辅助线仍只绘制在实际图片范围内；指针位于底板时不把辅助线延伸到底板，可在图片边缘显示钳制后的吸附反馈。
- 鼠标进入实际图片区域后，始终显示穿过当前指针位置的水平和竖直辅助线；辅助线裁切在图片边界内。该显示不依赖当前工具、是否正在画框或是否选中矩形；默认选择模式、没有任何框被选中、一次性画框结束并退回选择模式、框移动/缩放、中键平移和其他画布操作期间都持续跟随指针。
- 指针离开实际图片、没有图片或图片加载失败时隐藏辅助线。中键平移不再是隐藏条件；平移过程中图片边界发生变化时，辅助线应根据当前指针与移动后图片的交集实时重算。
- 鼠标中键按下并拖拽：平移图片；松开中键立即结束平移，不改变当前选择工具或矩形工具状态。
- 鼠标滚轮：竖直滚动图片；`Alt + 滚轮`：水平滚动图片。滚动到图片边界后停止，不允许图片无限漂离可视区域。
- `Ctrl + 滚轮`：向上放大、向下缩小；指针位于图片内时以指针对应的图片像素为缩放锚点，否则以画布可视区域中心为锚点。`Ctrl + Alt + 滚轮` 按缩放处理，不触发水平滚动。
- `Ctrl + 滚轮` 是固定鼠标手势，不进入键盘快捷键配置，但与缩放按钮、比例入口、`100%`、适配窗口和可配置键盘缩放动作共享同一视图状态；它不得创建标注历史、自动保存或复核状态变化。
- 图片支持面向像素级检查的 `1%–6400%` 手动缩放；适配窗口可低于 1%，放大/缩小按钮可连续到达整个范围。
- 大倍率缩放只能改变画布视图变换，不得生成按缩放尺寸展开的巨型图片副本。即使在 `6400%` 下，矩形原图坐标、鼠标命中、十字辅助线和自动保存结果也不得漂移。
- 高倍率下可通过中键平移、滚轮竖直滚动和 `Alt + 滚轮` 水平滚动到达图片各区域；框线、控制柄和标签文字保持可操作的屏幕尺寸，不随倍率变得粗大到遮挡目标。
- 800% 以下使用平滑插值，800% 及以上使用像素清晰模式；当前最大倍率固定为 `6400%`。
- 框线和内部填充均使用类别颜色。集中式令牌的外框 alpha 为 235、填充 alpha 为 38，外框清晰而内部不遮挡图片细节。
- 选中、悬停、归档和异常状态需要额外边线、控制点或纹理，不得只通过改变标签颜色表达。

> 状态：一次性矩形、边界滚动、高倍率映射、上下文系统指针、浅色底板、底板输入边缘吸附、默认选择模式持续辅助线与 `Ctrl + 滚轮` 锚定缩放均已实现并复验。工作台完整字标与首页保持同一 170px 可见宽度基准。

## 3. 数据集主页与数据集池体验

- 应用正常启动始终先进入数据集主页，不自动跳过主页打开上次的数据集。
- 主页顶部显示完整 DatumDock Logo、数据集搜索、设置和关于入口；主体以舒适的响应式卡片网格展示受管数据集，并包含醒目的“新建数据集”卡片。
- 数据集卡片至少显示名称、代表缩略图、图片数、标签数、复核进度与最后修改时间。异常数据集保留卡片位置并显示诊断入口，不让整个主页崩溃。
- 点击已有卡片直接打开该数据集的标注工作台；新建数据集成功后也直接进入工作台，并显示导入图片和管理标签的空状态引导。
- 工作台不显示工作区、项目树或外部目录。顶部品牌区提供返回主页入口，数据集下拉清楚显示并可快速切换当前数据集。
- 用户返回主页或切换到另一数据集时，不重新导入或同步扫描无关数据集；自动保存失败必须先处理，不能静默丢弃修改。
- 右侧下半区显示当前数据集的全部样本，并支持按来源、标注状态、标签和文件名筛选；左侧仅保留画布标注工具。
- 样本列表支持至少一万张图片，采用虚拟列表与延迟缩略图：首屏优先显示、滚动时按需加载，并在后台显示缩略图生成进度，不因全量加载卡住界面。
- 每个样本显示缩略图、文件名、标注数量，以及“待复核 / 已完成”两种图片级复核状态之一；无标注和异常分别作为数据条件与健康提示显示，不混入复核状态。
- 网格工具栏提供“显示标注预览”开关；开启时缩略图叠加矩形框、标签色和别名，关闭时只保留原图与状态标记。
- 顶部标题与主操作栏提供“导入图片”“导出”“切换数据集”“管理标签集”“管理模型”等入口；AI 标注位于左侧工具栏。所有操作目标都是当前数据集，“新建数据集”只在主页及主页全局菜单提供。
- “新建数据集”向导先选择“空数据集”或“从已有数据集创建”。模板模式显示将复用的标签集、命名规则和导入规范，并明确不会复制图片和标注。
- 数据集右键菜单提供“比较标签兼容性”“复制到数据集”“移动到数据集”“合并到此数据集”。不兼容目标显示差异入口而非执行按钮。
- 导入向导允许选择 JPG/JPEG、PNG、BMP、WebP、TIFF 等常见静态图片；MVP 统一转换为 PNG，并显示格式、数量、重复项、失败项和可取消的后台进度。
- 导出采用一次性分步向导：选择目标训练模型/格式 → 选择候选范围 → 设置比例与种子 → 预览统计 → 选择输出位置 → 执行并展示结果。
- 切换模型格式时，向导动态显示该格式所需参数、支持的标注类型和输出示例；不支持的样本或标注要在执行前明确提示。
- 导出前预览显示 train/val/test 的样本数、标签框数量和相似组约束造成的比例偏差；存在问题时明确提示，而不是自动忽略。
- 数据集工具栏提供“相似图片检查”入口，显示高置信度相似组和待确认候选；用户可确认相似组、移出误报图片或忽略候选，图片不会被自动删除。

### 3.1 归档与永久删除数据集

- 数据集卡片主体点击只负责打开数据集；“重命名”“归档”和“永久删除数据集”位于卡片右上角“更多”菜单。永久删除使用危险色图标和文字，但不在卡片悬停时成为默认焦点。
- 用户选择永久删除后先进入影响预览页，显示数据集名称、UUID 摘要、图片、标注、标签、模型、样本回收站、文件数、占用空间和当前后台任务。无法完成统计时只允许取消或打开诊断。
- 下一步要求输入完整数据集名称。输入框旁持续显示不可撤销说明；匹配前“继续”按钮禁用，粘贴和输入均允许，名称内容不被翻译。
- 最终确认页再次显示名称、图片数和空间，主按钮明确写为“永久删除数据集”，不能只写“确定”。取消始终为默认安全操作，按 `Esc` 关闭且不产生任何文件变化。
- 当前标注保存失败或存在写入任务时，对话框不允许越过保护；提供“返回处理保存”“查看任务”或“取消删除”，不自动放弃编辑或强制终止原子操作。
- 删除执行中显示阶段、进度和错误详情。进入文件暂存后窗口关闭不会假装取消；任务中心继续显示状态，失败时显示恢复/清理位置和下一步。
- 完全成功后返回主页并移除卡片；其他数据集的排序、筛选和当前内容保持不变。外部来源、导出目录和备份包在影响列表中明确标记为“不会删除”。
- 首版不提供整数据集回收站；需要可恢复隐藏时使用归档。样本级回收站仍只管理单张/批量图片，不承担整数据集恢复。

### 3.2 首页快速开始与学习中心

- 首页除数据集卡片外，提供“快速开始”和“学习中心”，让第一次使用数据集工具和 YOLO 的用户不必先阅读仓库文档。
- 首次启动时，“快速开始”位于数据集区域上方，以五个可点击步骤展示：创建数据集 → 配置标签 → 导入图片 → 标注/审核 → 导出训练数据。每一步显示一句目的说明，并可直接跳到对应页面或打开相关教程。
- 用户完成某一步后显示轻量进度，不强制按顺序操作。用户可折叠或关闭快速开始；关闭后仍可从学习中心、帮助菜单和设置页重新打开。
- 非首次启动且已有数据集时，数据集列表仍是主页最重要内容。学习中心放在数据集卡片下方或右侧辅助区，不能用大幅轮播、自动播放视频或弹窗遮挡用户进入数据集。
- 学习卡片至少包括“DatumDock 快速入门”“标签与矩形框标注”“AI 标注与人工审核”“YOLO Detection 入门”“数据划分与数据泄露”“导出 YOLO 并开始训练”“X-AnyLabeling 互操作”“备份与恢复”和“常见问题”。
- 点击卡片在应用内教程阅读器打开。阅读器包含目录、上一步/下一步、预计阅读时间、完成标记和返回当前操作的入口；教程中可使用静态截图、短动图或交互式控件高亮，但不得自动播放声音。
- DatumDock 教程覆盖从创建数据集、配置标签、导入与去重，到矩形框操作、图片级状态、AI 复核、YOLO 导出和备份恢复的完整闭环。
- YOLO 教程至少解释：检测任务是什么、图片与标签文件关系、`class_id x_center y_center width height`、归一化坐标、`data.yaml`、train/val/test、相似图片导致的数据泄露、导出前检查、训练环境与 GPU/CPU 的基本区别，以及常见目录或类别错误。
- “导出 YOLO 并开始训练”必须明确 DatumDock 当前负责数据管理、标注和数据集导出，不在 MVP 内直接执行模型训练。训练命令和第三方界面可能随版本变化，教程应标注适用版本，并把“打开官方文档”显示为外部链接。
- 核心教程随安装包离线提供，在没有网络时仍可阅读。外部链接必须带外链图标和目标说明，打开失败不影响本地教程。
- 教程正文和图片说明支持简体中文与英文即时切换；切换语言保持当前章节和阅读进度，不翻译或修改任何数据集内容。
- 教程完成进度属于全局应用偏好，不写入数据集、备份或 X-AnyLabeling 交换目录。应用升级时保留进度；教程版本变化时只把新增章节标记为未读，不清空旧进度。
- 首页可显示当前应用版本对应的“新功能”入口，但不得请求账号、展示广告、加载未经用户同意的远程资讯或影响离线启动速度。

## 4. 视觉风格：现代、轻快、友好且专业

完整设计令牌、组件语言、禁止项和截图验收以 [现代视觉设计规范 v2](VISUAL_DESIGN.md) 为准。v2 覆盖旧的暖灰/灰绿莫兰迪主视觉，目标不是传统 Qt 管理软件，也不是儿童化界面：

- 首页借鉴 Scratch 的友好、圆润、明亮、图标化和新手可理解性，用更宽松的卡片与引导降低使用门槛。
- 标注工作台借鉴 X-AnyLabeling 的画布优先、工具紧凑、右侧信息密度和直接反馈，保持生产力工具效率。
- 统一使用 DatumDock 自有冷白/浅蓝灰表面、品牌蓝、浅橙/浅蓝 `DD`、冷调浅色画布底板和现代圆角组件，不复制任何第三方品牌、图标、代码或像素布局。
- 首页可以更活泼，标注页必须更克制；两者共享字体、间距、组件、状态和图标系统，不能像两个拼接的软件。

### 品牌使用

- 数据集主页顶部和标注工作台标题栏左端固定设置品牌区：主页和工作台在空间允许时均显示同一视觉尺寸的完整 `DD + DatumDock` 字标。Logo 是静态展示，点击没有任何效果；工作台在 Logo 左侧提供 44×44 逻辑像素的独立返回按钮。
- 工作台常规宽度（含 1366×768 目标窗口）下，左上角完整字标以 170 逻辑像素为基准，实际可见宽度保持在 160～180 逻辑像素，可见图形高度不低于 24 逻辑像素。验收按非透明可见图形测量，不能只看控件或图标画布尺寸。
- 左侧窄标注工具栏顶部使用浅橙色与浅蓝色的 `DD` 标记作为紧凑品牌锚点；鼠标悬停显示 `DatumDock` tooltip。窗口较窄时，标题栏完整字标可收敛为 `DD`，不得把文字挤压到难以辨认，也不得占用矩形框和 AI 标注的必要空间。

### 启动窗口状态（待实施）

- 普通启动和 `--ui-preview` 启动完成后，主窗口都自动最大化到鼠标所在或系统判定的当前显示器可用区域。
- 这里的“全屏”采用 Windows 最大化窗口语义：保留原生标题栏、最小化/还原/关闭按钮与任务栏，不进入无边框、置顶或独占显示模式。
- 多显示器环境只最大化到一个显示器，不跨屏拉伸；DPI 为 100%、125% 或 150% 时都按 Qt 逻辑尺寸计算可用区域。
- 最大化后用户仍可点击“还原”并手动调整窗口；本轮要求下次重新启动时再次默认最大化，不以此前的普通窗口尺寸覆盖启动默认值。
- 启动画面到主页或工作台的过渡不得先显示明显的小窗口再跳大；应在首次可见前确定最大化状态，避免闪烁。
- 对话框、快速标签小窗和向导不随主窗口最大化，仍按内容设置最小/建议尺寸，并限制在当前屏幕可用区域内。
- 首次启动空主页、数据集主页、新建数据集向导首屏与“关于 DatumDock”页面使用完整字标；普通内容页不重复放置大 Logo，避免挤占数据管理与标注空间。
- 所有应用内展示统一使用 `assets/brand/datumdock-wordmark-v3.png`；在 GUI 资源打包阶段将它纳入 Qt 资源。后续的小尺寸窗口图标、托盘图标和安装包图标从同一 `DD` 标记派生，不以文字截图替代。
- 品牌字标仅用于产品身份，不可作为危险操作、状态颜色、标签类别颜色或可点击的主要操作按钮；始终提供文字标题和无障碍名称，不让 Logo 成为唯一的导航线索。

### 页面密度与反馈

- 首页使用 24–32px 外边距、14–16px 卡片圆角和较大标题；数据集卡片、快速开始与学习中心主次明确。
- 标注工作台使用 6–12px 紧凑间距、8px 工具圆角和浅色底板；左侧工具与右侧列表信息密度高，但常驻正文不小于 13px。图片本体通过细边界与底板区分，不能依赖深色大面积背景制造对比。
- 主操作使用品牌蓝实心按钮；次操作使用白色边框或幽灵按钮；危险操作只在确认区域使用红色。
- 选中、hover、键盘焦点、待复核、已完成和健康异常同时使用背景/边框、图标和文字，不只依赖颜色；健康异常与复核状态可以同时存在但视觉层级不同。
- 动画时长以 120–180ms 为主，并尊重系统“减少动态效果”；画框、拖动、缩放、切图和快捷键反馈不得等待动画。
- 默认 Qt 灰色立体按钮、凹陷输入框、大面积米黄灰绿、密集分隔线和传统菜单堆叠视为视觉缺陷。

### 图标资产

- DatumDock 的产品图标、导航图标、工具栏图标、空状态插图和安装包图标均作为自有项目资产生成并保存在 `assets/icons/`；其中应用图标从品牌 `DD` 标记派生。不得直接复制 X-AnyLabeling 或其他产品的品牌标识、图标文件或视觉资产。
- 图标统一采用圆角、简洁线性、低饱和单色/双色风格，默认 20px 或 24px 视觉网格；需要强调时使用主题语义色，不使用与数据集标签混淆的颜色。
- 每个图标必须有清晰文字 tooltip、可访问名称和禁用/悬停/焦点状态。仅图标按钮在常用场景下仍须能通过 tooltip 和快捷键说明被理解。
- 首批至少覆盖：应用图标、主页、数据集卡片、返回主页、导入、导出、标签、模型、自动标注、筛选、搜索、设置、帮助、保存、撤销/重做、删除、回收站、上一张/下一张、缩放、适配画布和状态提示。
- 用户后续指定某个图标需要修改时，以对应源资产为唯一修改目标；更新后需生成目标尺寸并在亮色主题、悬停、禁用和高 DPI 下进行视觉回归。

### 图标接入完整性

- “资源文件存在”不等于“界面已完成”。每个需要图标的可见操作必须通过 `IconRegistry` 绑定语义名称，并在路由遍历和截图清单中验证实际显示。
- 工作台顶部的导入、分类导出、标签管理和模型管理使用“20/24px 图标 + 文字”；首页学习中心、版本说明、设置和关于页使用同一规则。窗口变窄时可以收纳文字，但不得同时丢失图标、tooltip 和无障碍名称。
- 数据集卡片菜单、图片治理、标签/模型管理、向导步骤、错误报告、危险删除和状态徽标使用与含义一致的自有图标；危险操作不能使用普通信息图标，图片删除与标注框删除必须保持不同语义。
- 不再使用 `⚙`、`⋯` 等字符或 emoji 代替已有正式 SVG；原生平台确有更合适的标准控件符号时，必须在视觉清单中记录例外原因。
- 当前实现状态：双色 `DD` 应用图标、完整字标、返回、左侧画布工具、设置、更多和列表/网格已接入；首页导航、工作台顶部四个主操作、数据集卡片和部分弹窗仍待补齐。完成前不得把图标系统标记为全面完成。
- 开发启动只负责窗口和任务栏图标。桌面快捷方式、开始菜单、安装目录、EXE 资源和卸载程序图标由 PyInstaller/Inno Setup 阶段生成并在干净 Windows 环境验收。

## 5. 应用语言与设置

- 主菜单或应用工具栏提供“设置”入口；设置页包含“界面语言”下拉框：`简体中文` 与 `English`，默认选中简体中文。
- 选择语言后立即刷新当前可见的主窗口、菜单、工具栏、状态栏、标签/模型管理页面和对话框；无需重启应用。
- 设置页提供当前语言效果预览或说明文字，帮助用户确认切换结果。
- 语言切换不得改变标签名称、中文别名、描述、数据集名、图片文件名或模型配置；这些内容始终按数据集原值显示。
- 标签选择器的“中文别名 / 英文训练名 / 双语”显示方式属于标注显示偏好，与系统界面语言独立设置。
- 设置页“数据管理”分区提供“回收站少量样本阈值”和“默认导出比例”。每项右侧使用圆形问号图标，悬停显示该设置的影响、建议值和示例。

### 快捷键设置

- 设置页的“快捷键”分区必须展示全部由 DatumDock 定义的键盘操作，按“数据集与文件、浏览、画布、标注编辑、标签、自动标注、导出、窗口与帮助”等分组；不得只展示一部分“常用”操作，也不得在具体页面中保留无法配置的硬编码应用快捷键。
- 每行显示操作名称、当前绑定、默认绑定和说明；可用搜索框按名称或当前按键过滤。
- 点击绑定字段后进入录入状态，用户直接按下组合键；界面显示标准化后的组合，例如 `Ctrl+Shift+S`。
- 发现冲突时在该行显示冲突操作，并提供“取消”“替换原绑定”“选择其他按键”；系统保留组合显示为不可用。
- 每个应用操作允许修改绑定或清空为“未设置”；修改后立即在所有已打开页面和弹窗中生效，并更新菜单、tooltip 和快捷键说明。
- 底部提供“恢复当前项默认值”“恢复分组默认值”和醒目的“恢复全部默认按键”。恢复全部前显示将变化的操作数量和预览；确认后以一次原子设置事务清除全部用户覆盖并恢复当前版本的出厂绑定。
- 恢复全部默认按键只影响快捷键配置，不改变语言、主题、回收站阈值、数据集内容或标签。保存失败时保持恢复前的完整配置并显示错误，不允许只恢复一部分。
- Windows 无法由应用覆盖的系统组合键、普通文本编辑按键和纯鼠标手势不属于 DatumDock 应用快捷键；界面应解释不可用原因。中键/滚轮手势是否允许单独自定义属于后续输入映射增强，不阻塞本项。

## 6. 快捷键

下表是当前已注册的出厂默认绑定，不是写死的操作规则。除平台保留组合外，所有 DatumDock 应用快捷键都可在设置中修改、清空或恢复默认。

| 快捷键 | 行为 |
| --- | --- |
| `Ctrl+O` | 导入图片或文件夹到数据集池。 |
| `Ctrl+S` | 立即重试保存当前图片标注（通常编辑后已自动保存）。 |
| `Ctrl+E` | 打开数据集导出向导。 |
| `A` / `D` | 出厂默认分别切换到上一张 / 下一张图片；可在设置中修改，恢复全部默认按键后恢复为此绑定。 |
| `R` | 启动一次性矩形创建；完成一个框或按 `Esc` 后返回选择/编辑模式。 |
| `Delete` | 当画布或右侧当前标注列表具有操作焦点时，删除当前选中的矩形标注；文本输入状态下只删除文字。 |
| `Ctrl+Shift+Delete` | 删除当前图片及其全部受管关联信息。 |
| `Esc` | 取消正在创建的标注或取消选择。 |
| `Ctrl+Z` / `Ctrl+Y` | 撤销 / 重做当前图片中的有效标注编辑。 |
| `S` | 确认当前图片已完成人工复核。 |

- `A` 与 `D` 仅在标注工作台的图片浏览上下文生效。搜索框、标签名称、描述或其他文本输入控件获得焦点时，字母按键用于输入文字，不得穿透触发切图。
- 切图必须经过当前图片的自动保存离开保护：最新版本保存完成后才切换；保存失败时保持当前图片并显示重试、放弃内存修改或取消，不得因快捷键静默丢标。
- 到达数据集首张或末张时不访问无效样本，并给出轻量边界反馈；是否允许循环切换由后续显示设置决定，不改变 `A / D` 的上一张/下一张语义。

### 图片复核操作

- 右侧当前图片区域提供常驻“确认已完成”按钮；它主要用于“检查后无需修改”的待复核图片。发生有效人工编辑时无需再点按钮，编辑保存成功后自动显示已完成。
- 同一操作注册为 `review.mark_completed` 并进入全局快捷键设置；出厂默认按键为 `S`，可改绑、清空或恢复默认。
- 确认操作表示用户已经检查整张图片的漏标、误标、标签类别和框位置并认为没有问题。它只改变图片级复核状态，不改动任何矩形内容。
- 人工编辑内容与自动完成状态在同一次标注保存中原子提交；编辑或状态保存失败时都保持待复核并允许重试，不得只在界面上伪装为已完成。

## 7. 标签集与标签选择器

- 标签集由当前数据集的“管理标签集”入口打开的独立页面维护；标注侧栏只负责选择和查看标签。
- 标签管理页面上方是标签总表，默认列为：颜色、英文训练名、中文别名、描述、同义词、类别 ID、状态、使用次数和操作；支持搜索、排序、筛选、添加、编辑、归档，并在每行提供“查看图片”操作。
- 每个活动标签以独立颜色色块显示。新建时自动分配未使用的高辨识度颜色；编辑颜色时使用调色板和颜色选择器，并显示与已有标签的冲突提示。
- 标签颜色一致用于画布框线和半透明填充、标签选择器色点、标签检查集合的标记和自动标注建议框。选中、错误和待确认状态仍使用额外的边线、图标或纹理，避免只依赖颜色。
- 选择表格行后，下方或右侧详情区展示完整描述、同义词、创建信息、使用范围和修改入口；表格应支持直接编辑安全字段。
- 标签选择器顶部始终有搜索框；支持用中文别名、英文训练名、同义词或描述关键词定位标签。
- 默认条目格式为“`中文别名` · `english_name`”，下一行显示简短描述；不应只显示英文名称。
- 选中条目后，右侧说明区显示完整中文描述、英文训练名、类别 ID 和同义词，帮助用户在相似物品间做判断。
- 当标签数超过常用屏幕可见范围时，提供“最近使用”“常用置顶”“全部标签”分组与键盘上下选择；所有标签仍能通过搜索立即到达。
- 双击画布中的矩形框时，打开独立的“更改标签”小窗，并预选该框当前标签。打开小窗本身不修改标注，也不触发自动保存。
- 小窗结构参考用户草图但使用 DatumDock 现有视觉语言：顶部是常驻搜索框和“新建标签”入口，中部是标签卡片区，底部是固定的“确认”“取消”操作区。
- 小窗允许用户拖动边缘或四角自由调整大小，并设置保证搜索框、至少一个完整标签卡片和底部操作可用的最小尺寸；不得放大到当前可用屏幕区域之外。本轮暂不决定是否跨会话记忆窗口尺寸。
- 标签卡片使用响应式流式网格：窗口变宽时同一行显示更多标签，变窄时自动减少列数并换行；标签卡片不得因缩窄而重叠或裁切。内容超过可视高度时只滚动中部标签区，顶部搜索和底部按钮保持可见，不提供横向滚动条。
- 每个标签卡片至少显示颜色、中文别名和英文训练名；空间充足时增加简短描述，空间不足时通过 tooltip 或详情区显示完整描述。当前标签和准备选择的标签都必须有勾选、边框或文字状态，不能只靠颜色区分。
- 鼠标移入画布矩形、标签卡片、候选项、右侧当前标注行或标签管理表格行时，使用轻微填充增强、浅蓝背景、细品牌边框或轻微阴影提供即时但克制的反馈；移出后恢复，不改变卡片尺寸、矩形坐标或文字位置。
- hover、键盘焦点和已选中状态必须能同时区分。已选中状态使用更明确的背景、勾选或左侧指示条，优先级高于 hover；键盘焦点使用清晰焦点环，保证不用鼠标也能知道当前候选。
- 归档、只读或禁用标签降低视觉强调并使用对应指针/说明，不显示虚假可点击 hover。悬停与焦点只改变界面绘制，不修改标注、不更新最近标签，也不产生自动保存、历史或复核变化。
- 搜索应在输入时即时过滤，覆盖中文别名、英文训练名、同义词和描述；无结果时显示“没有匹配标签”和“新建标签”入口。`Ctrl+F` 可聚焦搜索框，方向键可移动候选，`Enter` 选择/确认，`Esc` 取消并关闭。
- 点击“新建标签”可在该小窗内展开快速创建表单，或打开其上的轻量二级窗；字段、唯一性和颜色冲突规则必须与完整标签管理页一致，不能创建临时字符串标签。创建成功的新标签立即出现在候选区并成为准备选择项。
- 用户点击“确认”后才把准备选择的标签应用到当前矩形，并形成一个历史节点和一次自动保存；点击“取消”、关闭窗口或按 `Esc` 时，当前矩形保持原标签不变。
- 成功创建矩形或确认标签改派后更新当前数据集的最近使用标签。打开/搜索/浏览后取消、零面积框、失败保存和只读 shape 不更新；切换数据集时分别恢复各自的会话状态，不能把另一数据集的标签带入。
- “新建标签成功后再取消更改标签”时，新建的数据集标签继续保留；标签定义保存与矩形改派是两个独立事务，取消外层窗口只取消尚未确认的改派。
- 若标签被归档，已有标注仍能正确显示；新建标注默认不再提供该标签。
- 用户编辑英文训练名或类别 ID 时，弹出迁移确认对话框：显示旧值、新值、当前数据集内受影响的样本数和标注数；确认前不写入任何数据。
- 迁移进行中显示总进度、当前文件和失败信息；完成后刷新标签表、画布、筛选结果和当前标注文案。
- 点击“查看图片”进入标签检查视图，汇总当前数据集中含该标签的图片。
- 标签检查视图以网格或列表展示缩略图、文件名、目标标签框数量和标注状态；点击图片会打开其原始标注页并高亮目标标签框。
- 检查视图是筛选结果而非新文件夹或新数据集，关闭后不会创建副本；样本被删除或标签变更时列表实时刷新。

## 8. 文件命名与样本删除

- “整理文件名”打开当前数据集的命名规则对话框。用户可设置前缀、起始编号、补零位数、命名模板和是否保留原扩展名，并选择“全部样本”或“当前筛选结果”。
- 输入规则时实时显示旧文件名 → 新文件名预览，标出重名、非法字符、编号空缺和无法写入的文件；存在错误时“执行重命名”不可用。
- 确认重命名后显示进度；完成后文件列表、画布标题和统计数据立即使用新名称。用户无需手工处理同名 LabelMe JSON 或 JSON 中的图片路径。
- 标注页面的“删除当前图片”使用显著危险色按钮，并与删除标注框的 `Delete` 键区分。确认对话框必须列出图片名、标注框数和将被一起删除的关联项。
- 删除对话框根据设置中的少量样本阈值提供“移入软件回收站”和“永久删除”两种明确选项，并显示样本数量和受影响关联项；批量删除时允许直接选择永久删除。
- 删除成功后自动跳转到下一个可用样本；当前图片、配套 JSON、索引项、缩略图、缓存和待确认自动标注一并移入回收站或永久消失，标签集和其他图片不受影响。

## 9. 模型库与自动标注

- 当前数据集的“管理模型”打开独立模型管理页面。页面上方表格展示：模型名称、格式、文件大小、任务类型、输入尺寸、类别数、状态、导入时间、标签映射状态和操作。
- 用户可点击“导入模型”选择 ONNX、PT 等文件。导入进度包含复制、解析、校验和配置确认四个阶段；解析结果以可读方式展示，而不是静默完成。
- 自动探测成功时，配置页展示模型的输入尺寸、预处理、输出解码、类别表、置信度阈值、NMS 阈值及当前数据集标签映射；用户可确认或修改需要配置的字段。
- 不支持或探测不完整的模型显示明确原因、缺失字段和可采取的下一步；禁止提供看似可用但实际未知的“自动运行”按钮。
- 更新模型时，用户在同一模型条目中导入替换文件；新文件验证成功前，旧模型保持可用。删除模型前显示受影响自动标注来源提示，确认后只移除当前数据集的模型文件和配置。
- 在数据集页面选择“自动标注”后，用户选择已验证模型、处理范围（当前图片、全部图片、全部未标注图片）和阈值。预测框直接保存为带模型来源、置信度和图片级“待复核”状态的数据集标注；用户在画布上执行任意一次有效人工标注编辑并保存成功后自动变为已完成。检查后无需修改时，使用“确认已完成”按钮或其可配置快捷键。自动标注不能无提示覆盖人工标注。
- 推理前优先使用可用 GPU；首次只能使用 CPU 时弹出说明，提供“继续使用 CPU”和“查看 GPU 配置方法”入口。
- 双击矩形框直接打开标签切换控件，其他画布编辑手势以 X-AnyLabeling 的常见操作习惯为参考。

## 10. 数据集备份与数据集间转移

- 主页和当前数据集菜单提供“导出数据集备份”与“导入数据集备份”。备份向导明确说明模型二进制不会包含在数据集包内，需要单独传递；模型配置会随数据集保存。
- 导入备份时先显示格式和完整性校验进度；校验通过后才允许设置新数据集名称并迁入内部资料库，失败时显示原因且不创建数据集。
- 数据集操作菜单提供“复制到数据集”和“移动到数据集”。目标列表仅显示标签集完全一致的数据集；不一致数据集以禁用状态并显示原因。

## 11. 标签集与数据集合并

- 标签管理页面的“比较标签集”对话框使用双栏/表格展示：训练映射一致、显示信息不同、仅源存在、仅目标存在和阻断冲突。
- “合并标签集”按钮仅在没有阻断冲突时可用。用户选择目标标签集为主版本后，预览将补充的标签和将保留的显示信息。
- 数据集合并向导显示源/目标名称、样本数、标签训练签名、完全重复图片数和“复制/移动”选择；发现标签不兼容时禁止继续。
- 合并进度页显示已处理、跳过重复、失败和待处理数量；结束后刷新目标数据集网格、状态统计和标签检查结果。

## 12. 自动保存与待处理状态

- 标签定义和图片标注都采用“有效变更提交即保存”。标签管理中的新增、编辑、归档、恢复和安全显示字段修改，在合法表单提交后立即原子保存；矩形创建、移动、缩放、删除、标签替换以及实际改变文档的撤销/重做，在操作结束后立即提交一个不可变自动保存快照。正常保存不依赖切图、离开页面、关闭窗口或手动按 `Ctrl+S`。
- 拖动过程中只更新画布预览，鼠标释放时提交一次；搜索、选中、悬停、打开后取消标签窗口、零面积矩形或没有实际变化的释放不保存。状态栏从请求入队起显示“正在保存”，仅在文件与 SQLite 真实提交并回读验证后显示“已保存”。
- “待复核”只允许由模型自动标注写入。纯人工创建的标注在首次保存成功后显示“已完成”；待复核图片发生新增、移动、缩放、删除、撤销/重做或标签替换等有效人工修改时，标注与“已完成”在同一次保存中提交。
- 自动保存失败时，当前文档进入待处理状态；切换图片或关闭窗口时，提供“重试保存 / 放弃内存修改 / 取消”三个选择。
- 保存失败状态栏必须可以点击打开详情，显示失败类型、请求 ID、数据集/样本/shape、编辑类型、版本、摘要、标签修订、异常链和恢复状态。只有真实 `PermissionError` 才能显示权限原因；版本冲突和外部修改不得提供危险的直接覆盖重试。
- 实机连续切图保存问题定位期间，保存详情同时显示临时日志的绝对路径。普通模式日志只记录版本、摘要前缀、请求代号和提交阶段，不记录图片内容；预览模式不得创建日志。日志开关是临时诊断措施，不作为长期用户功能或保存成功条件。
- 用户选择另一张图片后，上一张图片的画布立即结束编辑并进入轻量加载状态；新图片与标注文档尚未同时就绪前，矩形、改派、删除、撤销和重做均不可触发。加载完成后一次性恢复画布与右侧标注列表，不能让用户在视觉残留上产生跨样本编辑。
- 快速标签窗口中新建标签后，外层候选列表与标签集修订号必须同步刷新，用户可直接确认改派，无需关闭并重新打开数据集。若其他窗口恰好并发修改标签集，则显示“标签集已发生变化”，要求刷新标签，而不是提示检查磁盘权限。
- `Ctrl+S` 始终可用于立即重试保存；成功后状态栏显示“已保存”，失败信息必须保留到用户处理。

## 13. 无内容状态

- 内部资料库为空时，主页显示“新建第一个数据集”并解释数据将由 DatumDock 自动管理。
- 数据集为空时，工作台显示“导入图片”和“管理标签”两个主操作，并解释外部来源不会被修改。
- 文件夹不含支持图片时，说明没有找到图片并列出支持的格式。

## 14. X-AnyLabeling 导入与交换

- 文件菜单提供“导入 X-AnyLabeling 标注目录”和“导出为 X-AnyLabeling 目录”。两项均显示格式说明链接/问号帮助：交换目录使用图片、仅为含 shape 图片生成的同名 LabelMe JSON 与 `labels.txt`；无 shape 图片不生成空 JSON。该交换目录不等同于 YOLO 训练集导出。
- 导入向导按“发现图片 → 配对 JSON → 可编辑矩形 → 已保留但不可编辑内容 → 错误/缺失文件”展示统计与可展开的文件级报告。用户必须能在导入前看出哪些 shape 不能在当前版本编辑。
- 打开含已保留 shape 的图片时，右侧属性面板显示兼容性提示和 shape 数量；已保留内容在画布中以低干扰样式显示（若能安全渲染），不能被误认为可编辑矩形。保存前不需要用户额外选择“保留”，这是默认且不可关闭的安全行为。
- 导出向导说明它会使用当前数据集标签的英文训练名，而中文别名仅用于 DatumDock 界面；完成页显示导出路径、图片/JSON 配对数量、保留 shape 数量及失败项，并提供“在资源管理器中打开”。

### 步骤五正式接入状态

- 普通模式顶部导入菜单区分“导入图片”和“导入 X-AnyLabeling”；导出菜单提供真实 X-AnyLabeling 导出。两个操作均进入数据集绑定的后台任务，切换数据集不会把迟到结果写入当前工作台。
- `rectangle` 支持传统两个对角点以及 X-AnyLabeling 4.x 的完整轴对齐四角表示。旋转、退化或不能证明轴对齐的四点内容只读保留并显示警告；工作台提供用户主动确认的已有数据检查/修复入口，应用升级不得自动改写真实 JSON。
- 导入对话框先展示配对与文件级问题，再集中处理未知标签和重复图片；未知标签默认只读保留，用户可以显式映射或创建真实数据集标签。取消只在单样本边界生效，结果明确区分成功、跳过、失败和取消。
- 导出对话框要求用户选择尚不存在的新目录，展示范围、空标注、兼容 shape 和阻断项；只有完整验证通过才出现最终目录。
- `dataset.import_xany` 与 `dataset.export_xany` 已进入 `ActionRegistry`，默认不绑定快捷键，用户可在设置页配置。
- `--ui-preview` 只演示流程，不打开文件选择器、不启动文件任务，也不写资料库或设置。

## English Summary

The UX treats every valid label-related submission as an immediate save boundary. Label cards, candidates, annotation rows, and label-table rows also provide subtle hover feedback, while selected and keyboard-focused states remain stronger and accessible. Hover changes only presentation and never updates data, recent labels, history, autosave, or review state.

The real-machine log identified a cross-sample race during asynchronous switching. A stale canvas is now read-only until the target image and annotation are accepted together, and a bounded temporary log remains available for confirmation. Logging never determines save success and remains disabled in preview mode.
