# DatumDock 产品需求文档（MVP）

## 0. 2026-07-19 产品方向调整：内部数据集资料库

DatumDock 的目标入口已调整为“主页 + 受管数据集存档”，完整决定见 [内部数据集主页与存档式管理方案](DATASET_LIBRARY.md)。该方案优先于本文件后续章节中仍存在的旧入口表述：

- 用户不再创建或打开工作区，也不选择项目根目录。
- 应用每次启动先进入主页，展示以前创建的数据集和“新建数据集”入口。
- 点击已有数据集或创建新数据集后，直接进入该数据集的标注与管理工作台。
- 用户可见层级收敛为“数据集”；旧的项目级标签、模型和数据集池能力改为当前数据集独立拥有。
- 所有数据由软件自动存放在 Windows 当前用户的内部应用数据目录，外部图片只作为导入来源并被复制进受管池。

步骤二已经把正式 GUI 入口重构为 `AppLibrary -> ManagedDataset` 并接入内部持久化；步骤三迁入真实受管图片池，步骤四迁入数据集级标签、矩形标注、LabelMe 自动保存与人工复核。旧 `Workspace -> Project -> Dataset` 代码仅保留作迁移参考，不再由正式入口调用。模型、导出、备份与完整目录互操作仍需按后续章节逐步接入。

## 1. 问题与目标

许多本地视觉数据集分散在多个文件夹中，且人工划分训练、验证、测试集容易重复、遗漏或污染。DatumDock 的 MVP 应提供一个像游戏存档列表一样的主页，让用户管理多个独立数据集；每个数据集将图片、标注、标签与模型配置统一纳入软件内部的受管数据集池，用户可浏览和标注样本，再一键导出可复现、可直接训练的 YOLO 数据集。

产品完成度按 `docs/X_ANYLABELING_BASELINE.md` 分级对标 X-AnyLabeling 的核心工作流和质量闸门。DatumDock 不承诺在 MVP 一次性实现所有标注任务；任何未支持的形状、格式或模型必须明确呈现。

## 2. 目标用户

- 需要整理、清洗和导出训练数据的个人开发者和算法工程师。
- 需要处理小到中型本地图片数据集的标注人员。

## 3. MVP 范围

### 必须实现

1. 启动后进入内部数据集主页，创建、登记、搜索、重命名、归档或快速打开多个独立数据集，不要求用户选择工作区或保存目录。
2. 点击已有数据集或完成新建后，直接进入该数据集的标注与管理工作台；空数据集显示导入图片和管理标签引导。
3. 将一个或多个图片文件夹导入目标数据集的受管数据集池；MVP 统一将常见静态图片转换为 PNG。
4. 在当前数据集池中浏览、搜索、筛选并查看每张图片的标注状态与基础信息。
5. 标注工作台采用固定四区布局：顶部标题与导入/导出/数据集切换操作栏、左侧矩形框与 AI 等标注工具、中央图片画布、右侧当前图片标注和带图片级状态的图片列表。
6. 鼠标绘制、选中、移动、缩放和删除矩形标注框；MVP 暂不提供其他 shape。
7. 为标注选择当前数据集标签集中的标签；标签具有稳定英文训练名、中文别名、描述、同义词和颜色。
8. 标签选择器支持按英文名、中文别名、同义词和描述搜索；大量标签时默认优先显示中文别名与简要说明。
9. 使用快捷键切换图片、手动重试保存、删除标注和取消当前操作。
10. 为每张图片读写同名 LabelMe JSON 文件。
11. 每次导出时使用全局默认或手动设置目标训练模型、该模型支持的格式、训练集/验证集/测试集比例、随机种子和输出目录。
12. 将当前数据集中的已选样本导出为目标模型可直接使用的目录与标注格式；MVP 首先支持 YOLO Detection 目录结构与 `data.yaml`。
13. 标注编辑完成后立即自动保存；写入失败时在切换样本、返回主页、打开其他数据集或关闭程序前给予明确重试、放弃或取消提示。
14. 用户可在数据集级模型管理页面导入、更新、删除自动标注模型，并将可用模型结果直接写入当前数据集为“待复核”标注。
15. 用户可为当前数据集设置文件命名规则并批量重命名池内图片；也可在标注页面永久删除不合格样本及其关联信息。
16. 用户可在应用设置中将系统界面即时切换为简体中文或英文。
17. 用户可在应用设置中查看、搜索、修改、清空和恢复全部 DatumDock 应用操作的快捷键，并可一键恢复所有出厂默认按键。
18. 用户可导出完整数据集备份包并在另一台电脑验证后导入；可在标签集一致的数据集之间复制或移动样本。
19. 用户新建数据集时可复用其他数据集配置；可比较并安全合并兼容的标签集和数据集。
20. 用户可导入 X-AnyLabeling 已标注的图片目录，并将数据集导出为可直接由 X-AnyLabeling 打开的目录。
21. 首页提供可折叠的五步快速开始和内置学习中心，离线提供 DatumDock 使用教程、YOLO Detection 基础、数据划分/数据泄露、导出训练准备与常见问题，并支持中英文切换。

### 暂不纳入 MVP

- 多边形、点、折线、旋转框和实例分割。
- 视频、3D、医学影像或多模态标注。
- 团队协作、账号、云同步、在线数据集。
- 数据增强、训练、模型评测和实验追踪。

## 4. 用户主流程

1. 用户在 DatumDock 主页打开已有数据集，或新建一个由软件内部管理的数据集。
2. 用户导入若干图片文件夹；软件检测完全重复图片，必要时展示对比确认，然后复制并统一转码为 PNG 到目标数据集的受管池并建立索引。
3. 软件显示顶部数据集操作栏、左侧标注工具、中央画布和右侧标注/图片列表；用户可从顶部快速切换数据集，也可返回主页查看全部数据集。
4. 用户在右侧筛选并选择待标注图片，通过左侧选择矩形框或 AI 标注工具，在中央画布创建或调整标注；右侧当前图片标注列表与画布选择同步。
5. 用户调整或删除标注；软件立即安全保存并显示结果。
6. 用户在当前数据集中打开导出向导，使用全局默认比例或临时调整 `train / val / test`、目标训练模型、随机种子和输出目录。
7. 软件校验当前数据集的样本与标签集，按选择的模型格式划分数据并生成可用目录和配置文件。

### 标注画布交互整改（需求整理中）

以下内容是下一轮统一修改的产品要求，当前代码尚未修改；全部细节确认前不得把这些条目标记为已经实现：

- 按 `R` 只启动一次矩形创建。完成一个有效矩形后立即回到选择/编辑模式，不持续保持连续画框状态。
- 用户既可以按下左键拖拽并释放完成矩形，也可以第一次单击确定一个角点、第二次单击确定对角点完成矩形；两种方式必须产生相同的坐标、历史和自动保存结果。
- 两次单击是分离的“起点—终点”操作，不使用系统双击事件。第一次单击后显示锚点、动态预览和当前宽高；第二点可位于任意方向，最终坐标自动规范化。
- 零面积的第二点不创建标注，用户可重新选择第二点或按 `Esc` 取消。双击已有矩形继续专用于打开快速更改标签小窗，不得被两次单击画框逻辑抢占。
- 指针位于实际图片区域时，画布显示与指针相交的水平、竖直辅助线；辅助线不得延伸到图片外或覆盖侧栏。
- 标签矩形使用“清晰外框 + 低透明内部填充”。外框的不透明度必须高于填充，图片细节仍应清楚可见。
- 按住鼠标中键并拖拽可平移图片；普通滚轮竖直滚动图片；`Alt + 滚轮` 水平滚动图片。
- 上述滚动映射实施后取代当前滚轮缩放行为。缩放继续保留按钮、比例和适配窗口入口；`Ctrl + 滚轮` 是否用于缩放暂不锁定。
- 图片缩放必须满足精细边界检查，暂定至少支持 `6400%`。用户可通过缩放按钮和比例输入到达高倍率，并可一键回到 `100%` 或适配窗口。
- 高倍率只能修改视图矩阵，不能把原图重采样成巨型内存副本；矩形坐标、选中命中、辅助线位置和 LabelMe 保存值在任意倍率下保持原图像素精度。
- 放大后用户可结合中键、滚轮和 `Alt + 滚轮` 到达图片任意区域。框线、控制柄和标签浮层的屏幕可读尺寸不得随倍率无限增大。
- `6400%` 以上是否继续放大、以及高倍率像素插值方式，作为统一实施前的视觉与性能复验项。
- `Esc` 取消未完成矩形并返回选择/编辑模式；取消不得写入 JSON、进入撤销历史或改变图片复核状态。

## 5. 数据集池导入、命名与样本删除

### 常见图片格式、统一 PNG 与重复检查

- MVP 支持常见静态图片格式：JPG/JPEG、PNG、BMP、WebP 和 TIFF；不支持或损坏文件应在导入报告中逐项说明，不得静默跳过。
- MVP 导入后统一转换为 PNG，保留原始像素与透明通道；转码只作用于复制进软件内部受管池的文件，外部源文件绝不被改写。
- 后续“保留原格式”或统一 JPEG 等策略属于后续增强，当前导入界面不应提供未实现的格式策略。
- 导入前按图片内容哈希检测完全相同图片。若发现重复，弹窗展示待导入图片和已存在图片的并排对比、原路径和目标数据集；用户可取消或选择继续，同时保留两张图片作为不同样本。
- 导入完成后显示总文件数、成功复制/转换数、格式分布、失败项和重复项；一千至一万张常见图片的导入必须保持界面可响应并显示进度与取消入口。

### 受管文件命名

- 数据集可设置命名规则，将受管数据集池内图片按统一规格批量重命名；原始导入来源文件永远不被重命名。
- 规则至少支持自定义前缀、起始编号、补零位数和保留原扩展名，例如 `factory_part_{index:06d}.jpg`。
- 用户可选择重命名全部样本或当前筛选结果。执行前必须预览旧文件名、新文件名、冲突和受影响文件数量。
- 重命名后，样本稳定 ID 不变；同步更新池内图片名、同名 LabelMe JSON、JSON 的 `imagePath`、样本索引、缩略图缓存和任何内部文件路径引用。
- 名称冲突、非法文件名、无写权限或未能同步的引用必须阻止执行并显示原因。批量操作应使用临时文件和可恢复步骤，不能产生一半已改名、一半未改名的状态。
- 数据集可保存当前命名规则作为其元数据，便于下次导入或再次整理时复用；规则本身不影响模型导出时的命名策略。

### 从标注页面删除样本

- 标注页面提供明确的“删除当前图片”操作，与“删除当前标注框”使用不同的按钮、图标和快捷键，避免误删。
- 删除前对话框列出将被删除的内容：受管图片、配套 LabelMe JSON、样本索引记录、缩略图/缓存、待确认自动标注和其他该样本的派生信息。
- 对少量样本，用户可选择“移入软件回收站”；回收站保留完整样本包并支持恢复或永久清空。少量样本阈值在全局设置中配置，并在阈值旁提供悬停问号说明。对大批量样本，用户可选择“永久删除”，确认后直接清理关联内容。
- 确认后以一个受管事务移动或删除所有关联内容，并刷新文件列表、筛选统计、标签使用次数和当前画布。
- 删除样本不会删除数据集标签、模型、其他样本或导入前的外部源文件；若删除过程中失败，应用必须报告失败项并保持或恢复一致状态。

## 6. 数据集标签集

### 标签集边界

- 每个受管数据集在 MVP 中拥有一套独立标签集及其训练类别 ID。
- 标签集在创建数据集后即可预先配置，不必等到导入图片或开始标注。
- 数据集可暂时没有使用全部已定义标签；复制、移动或合并样本时必须先比较源与目标标签的稳定映射。
- 标签在不再使用时应支持“归档”而非直接删除，以保护已有标注与历史导出。
- 每个数据集提供单独的“标签管理”页面，用户无需打开任何图片即可集中维护该数据集的全部标签。
- 每个标签可生成“标签检查集合”：按标签稳定 ID 查询当前数据集中包含该标签的全部图片，供用户集中核查。

### 单个标签字段

| 字段 | 示例 | 用途 |
| --- | --- | --- |
| 英文训练名 `name` | `paper_cup` | 写入 LabelMe 标签和 YOLO `data.yaml` 的稳定机器名称。 |
| 中文别名 `alias` | `纸杯` | 标注人员在列表、画布和筛选中优先看到的可读名称。 |
| 标签描述 `description` | `一次性或可重复使用的饮用杯，不包含杯盖。` | 说明该物品是什么，以及必要时的边界规则。 |
| 同义词 `synonyms` | `饮料杯, 水杯` | 用于搜索，不改变训练类别。 |
| 训练类别 ID `class_id` | `12` | YOLO 导出的稳定整数类别 ID。 |
| 颜色 `color` | `#8FA39A` | 数据集内唯一的标签颜色；用于画布、标签表、筛选和检查集合中的一致视觉区分。 |

### 标签选择体验

- 标签选择器每项显示：颜色、中文别名、英文训练名和一行截断描述，例如“纸杯 · `paper_cup` — 一次性或可重复使用的饮用杯”。
- 新建标签时自动从标签调色板分配未被活动标签使用的高辨识度颜色；用户可通过颜色选择器修改。
- 活动标签不得使用相同颜色。用户选择已被占用的颜色时，界面应提示冲突并要求选择其他颜色；归档标签的颜色可被新标签重新使用。
- 标签颜色在当前数据集内一致显示：标注框边线/填充、标签列表色块、筛选项、标签检查集合和自动标注待确认框均使用该颜色。
- 颜色仅用于增强识别，任何状态仍需同时显示标签名称或图标，不能只靠颜色传达关键含义。
- 当标签较多时，输入框可同时搜索中文别名、英文训练名、同义词和描述；搜索结果优先匹配别名和英文名。
- 鼠标悬停或选中标签时，在属性面板显示完整描述、示例说明和英文训练名。
- 标注框默认显示中文别名；设置中可切换为英文名或“中文别名（英文名）”。
- 支持将常用标签置顶、按首字母或标签组浏览；标签组属于后续增强，不阻塞 MVP 的可搜索列表。

#### 双击矩形的快速标签小窗（需求整理中）

- 双击画布矩形打开可调整大小的标签选择小窗，当前标签预选；右侧标注项双击可继续复用同一组件。
- 小窗由顶部即时搜索/新建标签、中部响应式标签卡片网格和底部固定确认/取消组成。窗口越宽，每行显示的标签越多；窗口缩窄后自动换行，溢出内容使用纵向滚动。
- 标签很多时，搜索必须覆盖中文别名、英文训练名、同义词和描述，并保持键盘可操作。没有匹配项时可直接进入快速新建标签。
- 快速新建产生当前数据集中的真实标签，执行与标签管理页相同的必填、UUID、类别 ID、训练名和颜色唯一性校验；成功后新标签成为当前候选。
- 切换矩形标签只有在用户确认后才写入标注、历史和自动保存队列；取消时矩形保持原标签。
- 本功能当前仅完成需求记录。弹窗尺寸是否跨会话记忆，以及新建标签后取消外层小窗时是否保留新标签，等待统一实施前确认。

#### 从右侧标注列表删除当前标注（需求整理中）

- 单击右侧“当前图片的标注”列表项后，该项和对应画布矩形同步选中；此时按 `Delete` 删除该矩形标注实例。
- 此操作不得删除当前数据集的标签定义，也不得影响其他图片中使用同一标签的框。数据集级标签归档或删除只能从标签管理流程执行。
- 列表删除与画布删除必须复用同一业务命令、撤销历史和自动保存路径；完成后同步更新画布、列表、框数、标签反向索引和图片复核状态。
- 文本输入框、搜索框或编辑表单获得焦点时，`Delete` 不得穿透到标注删除命令。待复核图片删除标注属于有效人工编辑，保存成功后自动变为已完成；删除最后一个框以“已完成 + 零框”表示无目标。

### 标签检查集合

- 在标签管理表中，每个标签提供“查看图片”操作；点击后打开当前数据集的标签检查视图，而不是复制图片到新目录。
- 检查视图展示所有包含目标标签的样本缩略图、文件名、该标签的标注框数量、标注状态和最近修改时间。
- 用户可在网格与列表视图间切换，点击样本可跳转到标注画布并高亮目标标签的标注框；返回后保持检查集合的筛选位置。
- 检查集合为实时虚拟查询，标签被新增、移除、迁移或样本被删除后结果自动更新；不得创建重复图片或额外数据集。
- 支持在检查视图内逐张导航，并按是否含其他标签、标注数量和复核状态等条件进一步筛选。

### 标签修改与历史标注同步

- 标签管理页面顶部以可搜索、可排序的表格展示当前数据集全部标签；至少包含英文训练名、中文别名、描述、同义词、类别 ID、颜色、状态和已使用标注数。
- 用户可在表格中直接编辑，或打开详情面板编辑单个标签的完整信息。
- 修改中文别名、描述、同义词和颜色后，当前数据集的标注 UI、筛选和画布立即反映新的标签定义。
- 修改英文训练名时，软件先显示受影响样本数和标注数；用户确认后，批量更新数据集标签集及所有受影响 LabelMe JSON 中的 `shape.label`。
- 修改类别 ID 时，软件提示这会改变以后所有模型导出的类别映射；由于导出不保存历史，用户确认后更新数据集标签集，下一次导出使用新 ID。
- 批量迁移必须先完成预检，再以可恢复方式写入所有文件。任一文件无法写入时，应用必须报告失败项并回滚已写入内容，或保留可继续恢复的迁移状态；不得静默地只修改一部分。
- 标签稳定 ID 永不因编辑而改变；内部标注引用该 ID，LabelMe 文件中的英文名称是可由迁移同步的兼容表示。

## 7. 自动标注模型库

### 模型范围与管理

- 每个数据集拥有独立的模型库；模型文件复制到该数据集的受管模型目录，与其他数据集隔离。
- 提供独立的“模型管理”页面。用户可导入、查看、验证、更新、移除当前数据集的模型，并看到模型名称、格式、文件大小、导入时间、任务类型、输入尺寸、类别数、运行状态和关联标签映射。
- 更新模型指以新模型文件替换同一模型条目并重新自动探测配置；替换前保留旧配置直到新文件验证成功。
- 删除模型只删除当前数据集受管模型文件和模型元数据；不得删除任何图片、人工标注或其他数据集内容。若未来记录了自动标注来源，删除后历史标注仍保留，只显示来源模型已移除。

### 导入与自动配置

- MVP 优先支持 ONNX；导入时读取模型图、输入输出名称、数据类型、动态/静态尺寸、形状和可用元数据（如类别名称）。
- `.pt` 首先支持已验证的 Ultralytics YOLO 检测 checkpoint 及其兼容的自训练 YOLO 模型；软件应读取可用类别、任务与输入信息。其他 PT 变体后续按适配器逐步支持。
- 导入后执行一次本地校验：检查模型可加载、输入形状可解析、输出结构符合已支持任务。必要时使用测试输入进行安全探测。
- 自动生成并展示运行配置：任务类型、推理后端、输入宽高、颜色通道/预处理、归一化、输出解码方式、置信度阈值、NMS 阈值、模型类别表和当前数据集标签映射。
- 自动探测的信息可由用户在确认前复核、补充或修正；无法可靠确定的字段必须标为“需要配置”，不能以臆测值自动运行。
- 模型类别名称和当前数据集标签集不完全一致时，提供逐项映射界面；自动标注仅写入映射成功且未归档的标签。

### 自动标注行为

- 用户从当前数据集的模型库选择一个已验证模型，对当前图片、全部图片或全部未标注图片运行预标注。
- 自动标注优先使用可用 GPU；未检测到可用 GPU 或首次使用 CPU 时，软件说明原因、给出内置配置指引，并允许用户继续 CPU 推理。
- 自动标注结果直接写入当前数据集，附带模型来源、置信度和图片级“待复核”状态；用户在画布中完成任意一次有效人工标注编辑并成功保存后，整张图片自动变为“已完成”。检查后无需修改时，通过“确认已完成”按钮或可配置快捷键完成。自动标注不得无提示覆盖人工标注。
- 纯人工创建的标注默认把图片置为“已完成”。有效人工编辑包括创建、移动、缩放、删除、换标签，以及实际改变标注文档的撤销/重做；只读浏览、选择、画布缩放/平移、搜索和取消弹窗不触发完成。
- 双击矩形标注框可直接打开标签切换操作；其他常见编辑交互以 X-AnyLabeling 的图片标注习惯为参考。
- MVP 自动标注优先支持检测矩形框，后续随模型格式和标注类型扩展分割等任务。

## 8. 数据兼容性

- X-AnyLabeling 互操作是 MVP 的明确能力，详细字段、目录与回归规则以 `docs/X_ANYLABELING_INTEROP.md` 为准。
- 导入支持递归扫描 X-AnyLabeling/LabelMe 图片目录及其同名 JSON。图片复制后统一转换为受管 PNG，外部图片和 JSON 永不改写。
- 导入 JSON 兼容 `version`、`flags`、`shapes`、`imagePath`、`imageData`、`imageHeight` 与 `imageWidth`。每个 shape 的 `label`、`points`、`shape_type`、`group_id`、`description`、`difficult`、`score`、`flags` 和 `attributes` 必须可读或被保留。
- MVP 将 `shape_type: "rectangle"` 映射为可编辑矩形框，并以两个对角点写回；`label` 对应当前数据集标签的英文训练名。
- 多边形、旋转框、圆、线、点及其暂未支持的扩展字段以兼容载荷原样保留，界面明确提示“已保留、当前不可编辑”。对含有这些内容的文件保存、标签迁移、重命名和导出时不得静默删除或降级。
- 用户可选择“导出为 X-AnyLabeling 项目目录”。输出为 PNG 图片、同名 LabelMe JSON 和 `labels.txt`；JSON 的 `imagePath` 必须指向该目录内图片，DatumDock 私有元数据不得写入 JSON。
- X-AnyLabeling 导出独立于模型训练格式导出，不生成或保存导出方案、导出历史；它不替代 YOLO Detection 导出。

### YOLO Detection 导出

导出目录必须满足以下结构，并保证图片与同名标签文件相对应：

```text
export-name/
├─ data.yaml
├─ images/
│  ├─ train/
│  ├─ val/
│  └─ test/
└─ labels/
   ├─ train/
   ├─ val/
   └─ test/
```

- `data.yaml` 写入 `path`、`train`、`val`、`test`、`names`。
- 每个 `.txt` 标签文件使用 `class_id x_center y_center width height`，坐标归一化至 `0..1`。
- 类别 ID 由当前数据集的固定标签集决定，不因单次导出中的样本缺失而改变。
- 划分比例默认建议为 `80 / 10 / 10`；三者之和必须为 100%。
- 同一随机种子、相同候选样本及相同配置必须产生相同的划分结果。
- 导出仅生成目标模型训练所必需的文件；MVP 的 YOLO Detection 导出不额外生成或保存 `manifest.json`、导出方案或导出历史。

### 多模型导出

- 导出向导的第一步为选择“目标训练模型/格式”，而非选择已保存的导出方案。
- 每个模型格式定义自身支持的标注类型、数据划分方式、目录结构、配置文件和校验规则。
- MVP 交付 `YOLO Detection`；界面和架构应预留后续增加 `YOLO Segmentation`、COCO、Detectron2 或其他训练格式的能力。
- 同一数据集可多次导出为不同模型格式；这些导出互不影响，也不写回数据集池。

## 9. 非功能性需求

- 单个数据集至少支持一万张常见静态图片的导入、浏览、搜索、筛选、标注、标签检查和导出；界面不能因全量样本、缩略图或标注预加载而失去响应。
- 在普通办公电脑上流畅浏览 4K 以下图片；图片列表和缩略图按可见区域延迟加载。
- 操作结果可预测，重要错误应以可读提示展示。
- 核心标注读写逻辑应有自动化测试。
- 默认离线运行，不上传图片或标注数据。
- 整体界面遵守 `docs/VISUAL_DESIGN.md` 的现代视觉 v2：冷白/浅蓝灰表面、清晰品牌蓝、Logo 浅橙/浅蓝、现代圆角组件和深色画布取代暖灰/灰绿旧式 Qt 观感。
- 数据集主页和标注工作台顶部标题栏展示完整 DatumDock 字标；工作台左侧窄工具栏及窄窗口模式展示 `DD` 标记并提供 tooltip。首次启动空主页、新建数据集向导首屏与“关于”页展示完整字标，其他内容页保持克制，不重复占用数据操作空间。
- 首页可参考 Scratch 的友好、圆润、图标化和引导体验；标注工作台可参考 X-AnyLabeling 的画布优先、直接操作和信息密度。两者均不得复制第三方品牌、图标、代码或像素布局；标签颜色与应用主题和系统语义颜色严格分离。
- 文字、选中、键盘焦点、错误、禁用、加载和复核状态必须有足够对比度，并使用图标/文字等非颜色线索；应尊重系统缩放与“减少动态效果”偏好。
- 模型导入、探测和推理默认在本机执行；不自动上传模型、图片或标签信息。

## 10. 已确认的产品决策

- 首发仅支持 Windows 10/11 x64；安装包使用 PyInstaller 与 Inno Setup 构建。
- YOLO Detection 默认导出已标注样本；导出向导可明确把未标注图片作为负样本加入。
- 矩形框编辑、标签切换和撤销/重做采用立即自动保存。写入使用原子替换；只有保存失败时才保留待处理状态并要求用户决定重试、放弃或取消。
- 当前发布目标为 MVP + X-AnyLabeling L2 核心标注工作流。ONNX 与受支持 Ultralytics YOLO 检测预标注属于已支持的本地能力，但未实现 COCO/VOC 等格式矩阵前不得声明达到 L3。
- 视觉方向采用“友好外壳 + 专业工作台”：主页更接近 Scratch 式清晰亲和，标注页更接近 X-AnyLabeling 式紧凑高效；最终必须呈现 DatumDock 自有设计，不是任一参考产品的换色复制。

## 11. 应用语言

- 应用系统界面支持简体中文（默认）与英文；语言选择位于全局“设置”页面，而不是数据集设置中。
- 用户切换语言后，主窗口、菜单、工具栏、对话框、状态栏、设置页和后续打开的窗口应立即使用新语言；若某个第三方控件无法热切换，应在同一操作中明确提示是否需要重新打开该页面。
- 系统界面文案包括按钮、提示、错误信息、帮助文本和空状态；统一由翻译资源维护。
- 数据集内容不是系统文案：标签英文训练名、中文别名、描述、同义词、文件名、数据集名称和模型元数据均不得因界面语言切换而被翻译或修改。
- 用户可将标签的中文别名与英文训练名同时显示；该显示偏好独立于应用 UI 语言，避免英文界面下丢失用户定义。

## 12. 快捷键自定义

- 图片浏览的出厂默认按键固定为 `A` 切换上一张、`D` 切换下一张；用户可修改或清空这两个绑定，“恢复全部默认按键”后必须恢复为 `A / D`。
- `A / D` 只在图片浏览上下文生效，文本输入获得焦点时不得触发切图。快捷切图继续执行自动保存和离开保护，保存失败时不能切走并静默丢失修改。
- 全局“设置”页面提供“快捷键”分区，按功能分组展示全部 DatumDock 应用操作的名称、当前快捷键、默认快捷键、作用域和说明；任何可由应用触发的键盘操作都必须进入统一注册表，不得散落硬编码。
- 用户可按操作名称搜索，并点击某个快捷键字段后直接按下新的组合键完成录入。
- 可配置范围完整覆盖数据集操作、图片导航、保存、画布缩放与平移、标注创建/编辑/删除、撤销/重做、标签选择、自动标注、导入导出、窗口导航、设置与帮助；后续新增带键盘入口的操作也必须自动进入设置页。
- 用户可以为应用操作录入新绑定或清空绑定；清空后该操作仍能通过菜单、按钮或其他可见入口到达。
- 修改快捷键后应立即作用于已打开的窗口，并保存为全局用户偏好；它不属于任何数据集，不随数据集备份导入/导出。
- 若新快捷键与另一操作冲突，界面必须列出冲突操作，要求用户取消、选择其他组合或明确替换绑定。
- 平台不可安全覆盖的系统保留组合应标为不可使用并说明原因；普通文本编辑行为和纯鼠标手势不作为应用 `action_id` 伪装成可配置快捷键。
- 支持“恢复此项默认值”“恢复当前分组默认值”和“恢复全部默认按键”。恢复全部前展示变化数量，确认后原子清除全部用户覆盖并恢复当前应用版本的出厂绑定；写盘失败时完整回滚。
- 恢复全部默认按键不修改语言、显示、数据管理设置或任何数据集内容。恢复后所有已打开窗口、菜单和 tooltip 立即刷新，无需重启。
- 快捷键在中文和英文 UI 中保持同一绑定；仅操作名称与说明翻译。

## 13. 导出比例与数据泄露防护

- 全局设置提供默认 `train / val / test` 比例，初始建议 `80 / 10 / 10`；用户每次导出时可覆盖该默认值，导出结束后不保存为数据集导出方案。
- 比例之和必须为 100%，导出前显示每个集合的样本数和标签框数，并对空集合、极少数标签或明显类别失衡提出警告。
- 数据泄露指训练集和验证集/测试集包含重复、近重复或强关联样本，例如同一物品连拍、同一场景只改拍摄角度的照片被分到不同集合。模型可能记住背景或物品细节，导致验证/测试分数虚高，却无法反映真实新数据的效果。
- 当前版本对完全重复图片和已确认近似图片组执行分组划分：同一组必须整体进入一个 train/val/test 集合。划分器以“组”为最小单位，在不拆组的前提下尽量接近用户设定比例。
- 近似检测使用保守阈值先生成候选相似组；高置信度组可直接采用，边界候选必须在相似图片检查界面由用户确认、拆分或忽略，避免把只是背景相似的不同图片错误绑定。
- 近似图片不会自动删除、合并或减少样本数；其用途是避免导出数据泄露并帮助用户发现连拍或高度相似内容。

## 14. 图片网格、筛选与复核

- 标注工作台顶部固定提供当前数据集名称与快速切换、导入图片和导出入口；切换数据集前必须完成当前自动保存，失败时阻止静默切换。
- 左侧为画布工具栏，至少包含选择/编辑、矩形框、AI 标注、平移、适配窗口、撤销和重做；图片列表不占用左侧工具区。
- 右侧面板上半区显示当前图片的矩形标注列表，下半区显示图片列表/网格。标注项与画布框双向选中，图片项末尾固定显示文字和图形组成的状态徽标。
- 数据集池支持网格视图和列表视图；网格使用缩略图小窗快速选择图片，列表适合查看文件名、状态和统计字段。
- 网格工具栏提供“显示标注预览”开关。开启时，在缩略图上叠加已保存矩形框和标签颜色；关闭时只显示原图缩略图、复核状态和必要标记。该偏好可按数据集保存并随模板复用。
- 用户可按“待复核 / 已完成”两种图片级复核状态筛选，也可按无标注数据条件、健康异常、包含某个标签、文件名和导入来源筛选。
- 用户可见复核状态只有 `pending_review`（待复核）和 `completed`（已完成）。未标注是由框数/标注文件推导的数据条件，损坏是健康诊断，两者都不属于复核状态。
- 模型产生或改变标注后状态为待复核；复核时用户只要实际修改一次标注文档并保存成功便自动完成。若检查后无需修改，则点击“确认已完成”或使用其可配置快捷键。
- 纯人工画框默认已完成。没有任何目标框的图片也可由人工明确确认成已完成；不再显示单独的“已完成（无目标）”状态，零框事实用于识别负样本。
- 图片级状态、筛选、标签检查集合和网格标识应实时反映当前保存的标注内容。
- 不再提供未复核、待审核、有问题或已完成（无目标）等额外复核状态。待复核图片没有修正完时继续保持待复核。

## 15. 数据集备份与数据集间转移

- 用户可按数据集导出压缩备份包，至少包含数据集元数据、标签集、索引、图片与标注、可选缩略图缓存，以及模型配置；模型二进制始终不包含在备份包中，由用户单独传递和导入。
- 备份包包含格式版本、清单和完整性校验信息。导入前先校验结构、版本、清单与文件完整性；失败时不创建半导入数据集。
- 导入成功后数据集作为内部资料库中的独立受管副本保存，可在另一台电脑继续管理和标注。
- 用户可在标签集完全一致的数据集之间批量复制或移动样本。移动会从源数据集移除池内样本；复制会保留源样本。若标签集 ID、标签稳定 ID、类别 ID 或英文训练名映射不一致，操作必须阻止并说明原因。
- 批量编辑图片状态、标签或标注属性不进入当前 MVP。

## 16. 数据集模板、标签集比较与合并

### 新建数据集并复用配置

- 新建数据集向导提供“空数据集”和“从已有数据集创建”两种入口。
- 从已有数据集创建时，用户选择源数据集作为模板；复制一份独立的标签定义、标签显示配置、命名规则、导入 PNG 规范和数据集视图偏好，但不复制图片、标注、模型二进制、回收站内容或自动标注结果。
- 复制的标签保留稳定 ID、类别 ID 和英文训练名，以便后续安全比较、复制、移动或合并样本；之后两个数据集的显示信息和标签定义可分别维护。
- 若源配置损坏或含标签映射冲突，向导必须显示差异并停止自动创建，不能生成半配置数据集。

### 标签集比较与合并

- 标签管理页面提供“比较标签集”和“合并标签集”入口。比较报告按稳定标签 ID、类别 ID、英文训练名、状态、中文别名、描述、同义词和颜色逐项显示相同、缺失与冲突。
- 自动兼容的最低要求是：稳定标签 ID、YOLO 类别 ID、英文训练名和活动/归档状态完全一致；别名、描述、同义词和颜色的差异也需在报告中显示，但不改变训练映射。
- 标签集合并仅在不存在训练映射冲突时执行；用户明确选择目标标签集为主版本后，可补充源标签集中目标缺失的兼容标签并保留来源记录。
- 同名但不同类别 ID、同类别 ID 但不同英文训练名、或同稳定 ID 但语义不同的标签为阻断冲突。当前 MVP 不自动猜测映射，需用户先手动统一标签后再合并。

### 数据集比较与合并

- 数据集操作提供“比较标签兼容性”和“合并到此数据集”。合并前必须展示两个数据集的标签签名、样本数、重复图片数和目标数据集。
- 只有标签集训练映射完全兼容时，才允许将源数据集的样本包复制或移动到目标数据集；复制保留源样本，移动在目标写入并校验成功后清理源样本。
- 合并过程中仍执行图片内容重复检查；同图样本由用户在对比对话框中决定保留或跳过。
- 合并不自动修改任何标签定义，不自动丢弃标注，也不自动将不兼容数据集混合。

## 17. 首页快速开始与内置教程

## English Summary

DatumDock is a local-first dataset management and annotation desktop application. In the target two-state review model, the first effective manual annotation edit atomically changes pending review to completed when its save succeeds. A button or configurable shortcut completes review when no edit is needed; view-only actions do not count. No code has been changed yet.
