# DatumDock MVP 验收标准

所有项目都必须可以实际操作或由自动化测试验证。

当前对标等级和对应质量闸门见 `docs/X_ANYLABELING_BASELINE.md`。除本文件的功能验收外，声明达到 L1、L2 或 L3 时必须满足该基线中对应等级的全部必需项。

## A. 启动与界面

- [ ] 在目标平台执行项目说明中的启动命令后，应用正常打开。
- [ ] 窗口包含数据集池、图像画布、类别/属性区域、主操作入口和状态栏。
- [ ] 未打开项目及空数据集池均有明确的空状态提示。
- [ ] 界面使用一致的莫兰迪主题，且选中、错误和禁用状态可清晰区分。
- [ ] 默认浅色界面具有小清新、安静且专业的视觉层级：暖灰背景、低饱和面板、克制圆角和轻微分隔；页面不使用大面积高饱和色、厚重阴影、强渐变或密集常驻操作按钮。
- [ ] 主窗口、数据集网格/列表、标签表、表单、导出向导和确认对话框均复用同一组设计令牌、间距和文字层级；项目标签颜色不替代系统成功、警告、错误或选中状态。
- [ ] 在默认系统缩放与常用高 DPI 缩放下，文字、图标、状态角标和点击目标不重叠、不被裁切；键盘焦点、加载、未保存、错误和禁用状态均不只依赖颜色表达。
- [ ] 核心导航、导入导出、标注画布、模型、设置、删除/回收站和状态图标均使用项目自有且风格统一的资产；仅图标按钮具有 tooltip 和可访问名称，发布包中的应用图标在开始菜单和安装目录显示正常。
- [ ] 设置页支持简体中文与英文，默认简体中文；切换后当前可见系统界面文案立即更新，无需重启。
- [ ] 切换界面语言不会改变项目标签、描述、数据集名、文件名、标注内容或模型元数据。
- [ ] 设置页可搜索并修改常用操作快捷键；修改后无需重启即可在当前窗口生效。
- [ ] 设置页会阻止冲突和系统保留组合，或在用户确认替换时明确取消旧绑定。
- [ ] 用户可恢复单项、分组或全部默认快捷键；快捷键设置不保存进任何项目或数据集。

## B. 数据集池与图片浏览

- [ ] 用户可创建或打开本地工作区，并在其中创建至少两个项目和两个数据集。
- [ ] 用户新建数据集时可选择已有数据集作为模板，复用兼容标签集、命名规则和导入规范，但不复制图片或标注。
- [ ] 用户可快速切换项目与数据集；切换后只显示目标数据集的样本、类别和导出配置。
- [ ] 用户可导入至少一个图片文件夹到当前数据集的受管数据集池。
- [ ] 用户可导入一个 X-AnyLabeling 标注目录；每张已配对图片及同名 LabelMe JSON 均进入受管池，矩形框可在 DatumDock 中继续编辑，外部源目录不被改写。
- [ ] 导入含多边形、旋转框、圆、线、点或 shape 扩展字段的 X-AnyLabeling JSON 时，应用明确显示当前不可编辑的内容已保留；保存、重命名和标签名称迁移后，这些内容不被静默删除。
- [ ] 可导入 JPG/JPEG、PNG、BMP、WebP、TIFF 等常见静态图片；MVP 统一转换为受管 PNG，外部源文件不被修改。
- [ ] 检测到完全相同图片时，导入对话框显示待导入与已有图片的对比和来源；用户可取消或明确继续并同时保留两份样本。
- [ ] 软件能列出支持的图片文件，显示其基本标注状态，并在重复导入时给出提示。
- [ ] 导入后的图片和标注位于目标数据集池中；不同数据集的池内容互不混用。
- [ ] 用户可配置当前数据集的命名规则，并在执行前看到全部旧名称、新名称和冲突的预览。
- [ ] 批量重命名后，图片、配套 LabelMe JSON、`imagePath`、索引和缩略图引用均保持一致，且样本稳定 ID 不变。
- [ ] 用户可从标注页面删除当前图片；确认后该样本的受管图片、标注、索引和派生信息全部被清理，其他样本、项目标签和外部导入来源文件不受影响。
- [ ] 少量样本删除可移入软件回收站并恢复；“少量”阈值可在全局设置修改并有悬停说明；大批量样本可在明确确认后永久删除，两个操作都显示影响范围。
- [ ] 单个项目导入至少一万张常见静态图片后，可分页浏览、搜索、筛选、打开图片、执行标签检查与导出，且界面不会因全量样本或缩略图加载而失去响应。
- [ ] 网格视图的标注预览开关可显示或隐藏缩略图矩形框与标签色；关闭时仍清晰显示原图和图片级状态。
- [ ] 能正确打开 PNG、JPG/JPEG 至少两种常见格式。
- [ ] 用户可通过 UI 和快捷键切换上一张、下一张图片。
- [ ] 图片可缩放、平移，且缩放后标注位置仍正确。

## C. 标注编辑

- [ ] 用户可为项目创建标签集；每个标签可配置英文训练名、中文别名、描述、同义词、稳定类别 ID 和颜色。
- [ ] 每个活动标签自动或手动获得项目内唯一、可辨识的颜色；应用拒绝活动标签之间的颜色重复。
- [ ] 用户可通过独立标签管理页面集中查看、搜索、筛选、添加、编辑和归档项目全部标签；页面顶部表格至少展示英文训练名、中文别名、描述、类别 ID、状态和使用次数。
- [ ] 同一项目下不同数据集使用同一标签集和类别 ID 映射。
- [ ] 标签选择器默认展示中文别名、英文训练名和简短描述，并可按别名、英文名、同义词和描述搜索。
- [ ] 同一标签的颜色在所有数据集的画布、标签表、标签选择器、筛选/检查集合和自动标注建议中一致；关键状态不只依赖颜色区分。
- [ ] 用户可在标签管理表中对任一标签打开项目级标签检查集合，查看该项目全部数据集中包含该标签的图片、所属数据集和目标标签框数量。
- [ ] 用户可从检查集合跳转至原始标注画布，且目标标签框高亮显示；检查集合不复制或移动任何图片和标注文件。
- [ ] 用户可在图片上创建矩形框，并关联到指定标签。
- [ ] 修改中文别名、描述、同义词或颜色后，所有数据集中的标签显示即时更新。
- [ ] 修改英文训练名时，软件在写入前展示受影响范围；确认后项目标签集和所有受影响 LabelMe 标注文件同步更新。
- [ ] 迁移中发生写入失败时，软件能报告失败文件并保持或恢复一致状态，不能静默只更新部分标注。
- [ ] 用户可选中、移动、缩放和删除矩形框。
- [ ] 编辑操作会正确显示未保存状态。

## D. 标注保存与读取

- [ ] 保存会为图片生成或更新同名 LabelMe JSON。
- [ ] 重新打开图片后，已保存的矩形框、类别与坐标保持一致。
- [ ] 损坏 JSON 不会使应用崩溃，也不会在未确认时被覆盖。
- [ ] 离开未保存图片时，用户可选择保存、放弃或取消。

## E. 自动标注模型库

- [ ] 每个项目具有隔离的受管模型目录和独立模型管理页面；用户可查看、导入、更新和删除该项目模型。
- [ ] 导入 ONNX 后，应用能展示探测到的输入输出、尺寸、可用类别和运行配置，且只有本地校验通过后模型才可用于自动标注。
- [ ] 导入受支持 PT 变体后，应用能识别并展示其配置；无法识别或无法安全运行的 PT 文件必须显示明确的不支持原因。
- [ ] 可导入并运行 Ultralytics YOLO 检测模型及兼容自训练 YOLO 模型；其他不兼容 PT 文件不会被猜测运行。
- [ ] 用户可查看并修改模型类别到项目标签的映射；未映射模型类别不会创建项目标注。
- [ ] 更新模型时，旧模型在新模型验证成功前保持可用；删除模型不删除图片、数据集或人工标注。
- [ ] 有可用 GPU 推理后端时自动标注优先使用 GPU；首次 CPU 推理时说明原因、允许继续，并提供应用内 GPU 配置指引。
- [ ] 自动标注结果直接保存，带模型来源、置信度和图片级待人工复核状态；用户可修改漏标、误标、标签和框位置后，将整张图片标记为已复核或有问题，且自动标注不会静默覆盖人工标注。
- [ ] “有问题”与“已复核”是互斥的图片级状态；用户完成问题处理并标记已复核后，有问题状态被清除。

## F. 模型格式导出

- [ ] 用户可在全局设置设定默认 train/val/test 比例，并可在每次导出时选择当前数据集池样本范围、目标模型/格式、临时覆盖比例、随机种子和输出目录。
- [ ] 用户完成或取消导出后，应用不保存导出方案、导出路径或导出历史。
- [ ] 软件阻止比例和不为 100% 的导出，并对空划分给出清晰提示。
- [ ] 在输入样本和种子不变时，重复导出得到相同的样本划分。
- [ ] 当完全重复图片被划分到不同训练/验证/测试集合时，导出前会显示数据泄露警告。
- [ ] 已确认近似图片组中的所有样本在导出时被强制分配到同一个 train/val/test 集合；若组约束使比例偏差，导出预览会解释原因。
- [ ] MVP 的 YOLO Detection 导出目录包含 `images/train`、`images/val`、`images/test`、对应 `labels` 子目录与 `data.yaml`。
- [ ] 每个 YOLO 标签行符合 `class_id x_center y_center width height`，且后四项处于 `0..1`；`class_id` 与项目标签集映射一致。
- [ ] 导出不会修改或移动数据集池中的原图与原标注。

## F.1 X-AnyLabeling 双向互操作

- [ ] 用户可从当前数据集导出一个独立 X-AnyLabeling 目录，其中含 PNG 图片、同名 `.json` 和 `labels.txt`；每个 JSON 的 `imagePath` 指向同目录图片，尺寸与图片实际尺寸一致。
- [ ] DatumDock 产生的矩形框以 LabelMe `shape_type: "rectangle"`、英文训练标签名和两个对角点写入；DatumDock 私有的样本 ID、复核状态、模型来源和项目元数据不写入交换 JSON。
- [ ] 从 X-AnyLabeling 导入后未经编辑即导出，以及编辑 DatumDock 矩形框、重命名训练标签后再导出，均能保留未支持 shape 和其兼容字段；发生冲突或解析失败时给出文件级报告而非生成不完整目录。
- [ ] 自动化测试覆盖 X-AnyLabeling 样例目录的导入、矩形编辑、未支持 shape 保留及再导出；人工回归用已安装的 X-AnyLabeling 打开导出目录并验证图片、标签和矩形框可见。

## G. 质量

- [ ] 标注模型、LabelMe 序列化、数据集划分与 YOLO 导出逻辑具有自动化测试。
- [ ] 测试可重复运行并通过。
- [ ] 能完成一个包含至少 100 张普通图片的数据集浏览与逐张保存流程，无应用崩溃。

## H. GUI 与安装包交付

- [ ] 创建项目、导入数据、标注、筛选、导出等核心流程均可仅使用 GUI 完成。
- [ ] 发布构建可生成 Windows 安装包，且安装、卸载和开始菜单快捷方式均正常工作。
- [ ] 在未安装 Python 的干净 Windows 环境中，用户可安装并启动应用。
- [ ] 该环境中可以完成一次图片导入和 YOLO 导出，不依赖开发目录或终端。

## I. 项目备份与数据集转移

- [ ] 用户可导出项目压缩备份包；导入前应用校验项目格式、清单和完整性，校验失败不创建半导入项目。
- [ ] 成功导入备份后，项目中的数据集池、标签集、标注和索引可继续使用；模型配置保留，但模型二进制明确标记为需单独重新导入。
- [ ] 用户只能在标签集完全一致的数据集之间批量复制或移动样本；不一致时应用明确阻止并说明原因。
- [ ] 用户可比较两个标签集的训练映射和显示信息；只有无训练映射冲突时才可合并标签集。
- [ ] 用户可合并标签兼容的数据集；合并前看到标签签名、样本数和重复图片预览，合并后不丢失标注或留下半移动样本。

## J. X-AnyLabeling 对标发布闸门

- [ ] 发布说明明确当前完成等级（L1/L2/L3/L4）、已支持形状、格式、模型类型和已知限制。
- [ ] 声称“核心标注工作流可用”前，已完成 `docs/X_ANYLABELING_BASELINE.md` 的 L2 必需项。
- [ ] 声称“AI 与多格式工作流可用”前，已完成该基线的 L3 必需项。
- [ ] 未支持的功能不会以可点击但无效的入口、错误格式导出或静默降级的方式呈现。

## English Summary

Acceptance covers immediate Simplified Chinese/English system-UI switching and customizable, conflict-checked shortcut bindings without project-content mutation. It also covers workspace/project/dataset switching, isolated managed dataset-pool import and browsing, rule-based file renaming, complete managed-sample deletion, rectangle annotation, project-level label management, and live cross-dataset label inspection collections. X-AnyLabeling directories must round-trip: rectangles stay editable, unsupported shapes and metadata stay preserved, and the exported directory must open in X-AnyLabeling. A selected label's collection must open original samples and highlight matching shapes without duplicating data.
