> 审计范围：内部资料库、受管数据集、图片池、标签、矩形标注、双状态复核、快捷键、X-AnyLabeling/LabelMe 互操作和当前正式 GUI。模型推理、YOLO 导出、备份、跨数据集治理和安装包不属于本轮已实现范围。

## 3. 当前质量门

- Python 3.11、Ruff、Ruff 格式检查和 `compileall` 是本轮硬门槛。
- 测试清单当前为 267 项；加入步骤六导出回归后完整结果为 `266 passed、1 skipped`，切图加载期旧画布锁定、跨样本请求拒绝与第二行标签改派选择保持继续通过，且无 DatumDock 警告。自动回归通过不能替代用户原数据集的最终实机复验。
- Windows 符号链接安全用例因当前账户缺少权限可在本机跳过，但已加入 Windows CI；在具备权限的运行器上不得跳过。
- `pytest-cov` 已加入开发依赖和 Windows CI。当前本机从可信 PyPI 安装时发生 TLS EOF，未关闭证书校验；恢复后必须生成分支覆盖率报告，并按核心文件事务/标注/互操作模块 90%、其他当前生产模块 85% 的目标继续补测试。
- 全部自动化测试使用临时绝对 `DATUMDOCK_DATA_DIR`；预览模式必须零写盘，外部来源保持只读。
- 普通模式和 `--ui-preview` 均在 Windows 原生 Qt 后端进入事件循环；普通临时资料库生成 2 个初始化文件，预览模式生成 0 个文件。
- 本轮原生视觉证据位于忽略提交的 `build/ui-review/stabilization-native/`：步骤五双语三分辨率 24 张，近期整改双语页面 6 张；抽查确认中文字体、英文长文案、语义图标和危险删除对话框无本轮已知裁切。
- 真实 `%LOCALAPPDATA%\DatumDock` 仅做只读采样；最终前后均为 23 个文件，按“相对路径 + 制表符 + 文件 SHA-256”生成的树摘要均为 `BEFFB245757702C2C0659E6CBD443A8D0BB62A7A25CA091B30F6DD8293B2B0FF`。
